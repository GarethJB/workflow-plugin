# 홈 (메인페이지) 기획서

> Figma 소스: AI 식단 분석 서비스 MVP / 스토리보드 - 메인페이지
> 작성일: 2026-04-03

---

## 1. 개요

온보딩 또는 로그인 완료 후 진입하는 앱의 메인 화면. 선택한 날짜의 식단·영양소·수분 섭취 현황을 표시하고, 식단 등록/삭제/상세 조회를 수행한다.

**화면 구성:** 캘린더 → 대시보드 (3페이지 스와이프) → 식단 리스트 + FAB

---

## 2. 페이지 전역 정책

| 항목     | 정책                                                           |
| ------ | ------------------------------------------------------------ |
| 스크롤    | 상/하 스크롤 지원                                                   |
| 데이터 기준 | 디바이스 일자 기준 데이터 호출                                            |
| 숫자 표기  | 페이지 내 모든 숫자는 **정수**로 표현 (원데이터가 소수점일 경우, 소수점 첫째 자리에서 **반올림**) |
| 천 단위   | 만/천 단위에서 콤마 표시                                               |
| FAB    | 우측 하단 **항상 고정** 표시                                           |


### 2.1 데이터 갱신 시점

| 트리거                  | 동작       |
| -------------------- | -------- |
| 식단 등록 및 삭제 직후        | 즉시 반영    |
| 디바이스 기준 날짜 변동 시      | 데이터 새로고침 |
| 사용자 탭 이동 시           | 데이터 새로고침 |
| 앱 백그라운드 → 포그라운드 전환 시 | 데이터 새로고침 |

## 3. 캘린더 영역

### 3.1 날짜 표시 (최상단)

| 항목 | 정책 |
|------|------|
| 당해 날짜 표기 | `M. D 요일` (예: `2. 5 수`) |
| 지나간 연도 날짜 표기 | `YY. M. D 요일` (예: `25. 2. 5 수`) |

### 3.2 오늘 이동 아이콘

| 항목 | 정책 |
|------|------|
| 출력 조건 | 오늘 외 날짜의 데이터를 보고 있을 때만 노출 |
| 동작 | [오늘] 아이콘 터치 시 당일 날짜로 이동 및 데이터 새로고침 |

### 3.3 일자 선택 영역

| 항목     | 정책                                         |
| ------ | ------------------------------------------ |
| 스크롤    | 날짜 위 **좌/우 스크롤** 가능 (요일은 스크롤 고정)           |
| 과거 범위  | **2025년도부터** 표시                            |
| 미래 범위  | 당일 기준 해당 월이 포함된 주까지 표시                     |
| 미래 제한  | 오늘 이후 날짜로 터치(이동)는 **불가능**                  |
| 월 초과 시 | 바운스 효과                                     |
| 요일 표기  | 오늘 → "오늘" / 그 외 → 요일 (월, 화, 수, 목, 금, 토, 일) |

### 3.4 날짜 선택 동작

| 항목 | 정책 |
|------|------|
| 특정 날짜 선택 시 | 선택한 일자 기준으로 데이터 fetch (권장 영양소 + 식단 목록 + 수분 섭취량 병렬 호출) → 대시보드 + 식단 리스트 전환 |
| 식단 기록 有 | 디자인 구분 표시 (dot indicator) |
| 권장량 초과 | 권장 칼로리(100%) 초과 시 별도 컬러로 강조 |

## 4. 대시보드 영역

대시보드는 좌우 스와이프 **3페이지** 구성.

| 항목      | 정책                         |
| ------- | -------------------------- |
| 스크롤     | AOS: 좌우 스크롤 / iOS: 좌우 스크롤  |
| 날짜 변경 시 | 사용자가 보고 있던 대시보드 페이지 **유지** |

### 4.1 1페이지 — 칼로리 및 주요 영양소 (탄/단/지)

#### 4.1.1 일일 권장 칼로리

| 항목 | 정책 |
|------|------|
| 최초 | 가입 시 계산된 권장 칼로리 |
| 이후 | 사용자가 정보 수정 시, 재계산된 권장 칼로리를 **변경 이후부터** 반영 |
| **수정 이전 날짜** | 과거 날짜는 **수정 전 권장값 유지** (소급 적용 없음) |
| 가입 이전 날짜 | 가입 시점에 산정된 권장값 기준으로 적용 |

#### 4.1.2 반도넛 차트

| 항목 | 정책 |
|------|------|
| 시각화 | 권장 대비 섭취 칼로리 진행률을 반도넛 형태로 표시 |
| 채움 방향 | 좌측 → 우측 방향으로 채워지는 형태 |
| 100% 초과 시 | 게이지 **모두 채워진 형태** 유지, 초과 후 음식 등록 시마다 내부 영역별 재산정 |

#### 4.1.3 섭취 칼로리 표시

| 항목 | 정책 |
|------|------|
| 초기값 | 0 |
| 최대 자릿수 | 5자리 (0~99,999) |
| **99,999 초과 시** | **99,999 고정** 표시 |
| 소스 | 등록된 음식의 총 합계 칼로리 |

#### 4.1.4 칼로리 가이드 문구

| 조건 | 문구 |
|------|------|
| 식단 미등록 | "오늘은 어떤 음식을 드셨나요?" |
| 권장 칼로리 미만 | "권장 칼로리까지 {남은칼로리}kcal 남았어요" |
| 권장 칼로리 **정확히** 충족 | "권장 칼로리를 정확히 채웠어요!" |
| 권장 칼로리 초과 | "권장 칼로리를 {초과칼로리}kcal 초과했어요" |

#### 4.1.5 탄수화물 / 단백질 / 지방

| 항목 | 정책 |
|------|------|
| 표시 형식 | 섭취량 / 권장량 (g단위) |
| 섭취량 자릿수 | 최대 5자리 (0~99,999), 초과 시 99,999 고정 |
| 권장량 자릿수 | 최대 4자리 (0~9,999), 초과 시 9,999 고정 |
| 비율(%) | 전체 섭취 대비 각 영양소 비율 표시 |

**상태 라벨 기준:**

| 영양소 | 부족 | 적정 | 과다 |
|--------|------|------|------|
| 탄수화물 | < 80% | 80 ~ 120% | 120% 초과 |
| 단백질 | < 90% | 90 ~ 130% | 130% 초과 |
| 지방 | < 80% | 80 ~ 110% | 110% 초과 |

### 4.2 2페이지 — 부가 영양소

#### 4.2.1 제한 권장 부가 영양소

대상: **당류 / 나트륨 / 콜레스테롤**

| 항목 | 정책 |
|------|------|
| 표시 형식 | 섭취량 / 권장량 (단위: 당류 g, 나트륨 mg, 콜레스테롤 mg) |
| 막대 그래프 | 권장량 대비 0 ~ 1.5배 범위 내 표현, 구간별 영역 구분 |
| 1.5배 초과 시 | 게이지 최대치(Full) 고정, 수치는 실제 섭취량 표시 |
| 기준선 | 100% 지점에 기준선 표시 |

**상태 라벨:**

| 상태 | 조건 | 기본값 |
|------|------|--------|
| 적정 | 권장 섭취량 이하 | (기본값) |
| 주의 | 권장 섭취량 100% 초과 ~ 120% 이하 | — |
| 위험 | 권장 섭취량 120% 초과 | — |

#### 4.2.2 섭취 권장 부가 영양소

대상: **식이섬유**

| 항목 | 정책 |
|------|------|
| 막대 그래프 | 권장량 대비 0 ~ 100% 범위 내 표현 |
| 기준선 | 100% 지점에 기준선 표시 |

**상태 라벨:**

| 상태 | 조건 | 기본값 |
|------|------|--------|
| 부족 | 권장 섭취량(100%) 미만 | (기본값) |
| 적정 | 권장 섭취량(100%) 충족 | — |

### 4.3 3페이지 — 음수량

| 항목 | 정책 |
|------|------|
| 단위 | 리터(L), **소수점 둘째 자리**까지 표시 |
| 표시 형식 | 섭취량 / 권장량 (예: `0L / 2.5L`) |
| 상한값 | **5L** — 최대치 도달 이후 (+)버튼 **미출력** |
| 하한값 | **0L** — 최소치 도달 이후 (-)버튼 **미출력** |
| 과거 데이터 | 등록 및 수정 가능 |

**인터랙션:**

| 항목 | 정책 |
|------|------|
| (+) 버튼 | 0.25L(250ml) 증가 |
| (-) 버튼 | 0.25L(250ml) 감소 |
| 반영 | 버튼 터치 시 컵 수위 및 수치 **즉시 반영** |
| 햅틱 | (-), (+) 버튼 터치 시 **햅틱 추가** (0L/5L 경계 도달 시 버튼 미노출이므로 햅틱 미발생) |
| 연속 터치 | 빠른 연속 터치 시 0.25L씩 정확히 증감, 각 터치마다 즉시 UI 반영 |
| API 호출 | 500ms 디바운스 후 저장 (연속 터치 시 마지막 값 기준 1회 호출) |

**컵 수위 표현:**

| 상태 | 컵 |
|------|-----|
| 권장 음수량 미달 | 섭취량에 비례하여 수위 채움 |
| 권장 음수량 도달 | 컵 수위 최대치 |
| 권장 음수량 초과 | 수치는 누적 표시, 컵 수위는 **최대치 유지** |

**보조 문구:**

| 조건 | 문구 |
|------|------|
| 음수 기록 없을 시 | "지금 물 한 모금 어때요?" |
| 음수 기록 있을 시 | "조금만 더 마셔도 좋아요" |
| 권장치 도달 시 | "오늘 필요한 물을 충분히 마셨어요" |

## 5. 식단 리스트 영역

### 5.1 리스트 구성

| 항목 | 정책 |
|------|------|
| 정렬 | **최근 등록순** (동일 시간 등록 시 **등록 선순대로** 정렬) |
| 등록 개수 제한 | 없음 (제한 정책 삭제됨) |

### 5.2 리스트 아이템 구성요소

| 구성요소 | 정책 |
|---------|------|
| 썸네일 | 식단 사진 / 사진이 없는 경우 대체 이미지 출력 |
| 등록 시간 | "오전/오후" + 12시간제 표시 (예: `오전 11:32`) |
| 식단명 (단일) | `{Food_Name}` |
| 식단명 (복수) | `{Main_Food_Name} 외 {Count-1}개` |
| 영양정보 | 칼로리(kcal) + 탄수화물(g) + 단백질(g) + 지방(g) |
| **데이터 정합성** | 리스트 각 항목의 합계 = 대시보드 표시값 (칼로리/탄단지 일치) |

### 5.3 리스트 비어있을 경우

| 항목 | 정책 |
|------|------|
| 대체 디자인 | 빈 상태 영역 + 문구 출력 |
| 문구 | "오늘 먹은 음식을 기록해 보세요" |
| 터치 시 | 식단 등록 프로세스 진행 (FAB 터치와 동일한 바텀시트 출력) |

### 5.4 식단 삭제

| 단계 | 동작 |
|------|------|
| 1 | 식단 블록을 **왼쪽으로 밀기** → 우측에 삭제 박스 출력 |
| 2 | 삭제 박스 외 부분 터치(or 스크롤) 시 → 닫힘 |
| 3 | 삭제 박스 터치 → **컨펌 다이얼로그** 출력 |

**컨펌 다이얼로그:**

| 항목 | 내용 |
|------|------|
| 타이틀 | "식단 기록을 삭제할까요?" |
| 서브타이틀 | "삭제한 기록은 다시 되돌릴 수 없어요" |
| 좌측 버튼 | "아니요" → 닫기 |
| 우측 버튼 | "삭제할래요" → 삭제 API 호출 → 데이터 갱신 |
| 폼 | 식단 상세페이지의 컨펌창과 동일 |

## 6. 식단 등록 (FAB)

## 8. API 기반 플로우

### 8.1 화면 진입 시

```
[화면 진입 (또는 데이터 갱신 트리거)]
  │
  ├─ ① POST /api/v1/daily/getCalendar
  │   Request: { stDate: "2025-01-01", enDate: "{today}" }
  │   Response: [{ theDate, nutrntEnergy, memGoalEnergy }]
  │   → 캘린더 dot 표시 + 권장 초과 여부 컬러 구분
  │
  └─ 선택 날짜 기준 병렬 호출:
      │
      ├─ ② GET /api/v1/member/getTdeeInfo?theDate={selectedDate}
      │   Response: { memGoalEnergy, memGoalChocdf, memGoalProtein,
      │              memGoalFat, memGoalSugar, memGoalNat,
      │              memGoalFibtg, memGoalSfa, memGoalTrnfa,
      │              memGoalChole, memGoalWater, ... }
      │   → 대시보드 권장량 표시
      │
      ├─ ③ POST /api/v1/food/head/list
      │   Request: { theDate: "{selectedDate}" }
      │   Response: { foods: [{ foodName, srvSize, srvUnit, srvCnt,
      │              nutrition, dietHdNo, dietDate, dietDtCnt,
      │              dietHdImgNm, dietHdImgUrl, dietHdNm }] }
      │   → 식단 리스트 + 대시보드 섭취량 합산
      │
      └─ ④ POST /api/v1/water/sel
          Request: { theDate: "{selectedDate}" }
          Response: { result: {waterAmount} }
          → 대시보드 음수량 표시
```

### 8.2 날짜 선택 시

```
[날짜 선택]
  │
  └─ 선택 날짜 기준 병렬 호출:
      ├─ ② GET /api/v1/member/getTdeeInfo?theDate={selectedDate}
      ├─ ③ POST /api/v1/food/head/list { theDate }
      └─ ④ POST /api/v1/water/sel { theDate }
      (상세는 8.1과 동일)
```

### 8.3 수분 업데이트

```
[(+) 또는 (-) 버튼 터치]
  │
  ├─ 즉시 UI 반영 (0.25L 증감)
  │
  └─ 500ms 디바운스 후:
      ⑤ POST /api/v1/water/ins
      Request: { theDate: "{selectedDate}", waterCnt: {currentWater} }
      Response: { code: "1" }
      → 연속 터치 시 마지막 값 기준 1회 호출
```

### 8.4 식단 삭제

```
[삭제 확인 (컨펌 다이얼로그)]
  │
  ▼
⑥ POST /api/v1/food/head/del
   Request: { dietHdNo: {삭제할 식단 번호} }
   Response: { code: "1" }
  │
  └─ 성공 시:
      ├─ 식단 리스트 갱신 (③ 재호출 또는 로컬 제거)
      ├─ 대시보드 수치 재산정 (섭취량 합산 갱신)
      └─ 캘린더 dot 갱신 (해당 날짜 기록 유무 재판별)
```

### 8.5 데이터 갱신 트리거

| 트리거 | 호출 API | 비고 |

| 날짜 선택 | ②③④ | 캘린더 dot은 유지 |
