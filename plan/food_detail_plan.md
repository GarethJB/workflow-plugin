# 식단 상세페이지 기획서

> Figma 소스: AI 식단 분석 서비스 MVP / 스토리보드 - 식단 상세페이지
> 작성일: 2026-04-03

---

## 1. 개요

식단 등록/수정의 핵심 화면. 모든 진입 경로(촬영/앨범/검색/기존 식단)가 수렴하며, 음식의 추가·삭제·수량 변경·영양정보 수정을 수행한다.

---

## 2. 화면 구성

### 2.1 상단 영역

| 항목 | 정책 |
|------|------|
| **타이틀** (식단 이름) | 단일 음식: 해당 음식명 / 복수 음식: 가장 고칼로리 음식명 + "외 N개". 음식 추가/삭제/**칼로리 수정** 시 고칼로리 기준 **재산정** |
| **더보기 버튼** | 터치 시 버튼 리스트 출력 |
| **삭제 메뉴** | 기록 완료 전 **비노출**, 기록 완료 후에만 노출 |
| **식단 사진** | 업로드된 사진 출력 + 분석 결과에 따른 라벨링 표시. **사진 없을 경우 대체 이미지** 출력 |
| **등록 시간** | 식단 등록 시간 표시 |

### 2.2 영양정보 요약 (수정 불가)

| 항목 | 정책 |
|------|------|
| 표시 내용 | 해당 식단의 **총 칼로리** + **총 영양소 (탄/단/지)** |
| **탄/단/지 게이지** | 탄/단/지 비율에 맞춰 **게이지 길이 연동** 표시 |
| 수정 가능 여부 | **수정 불가** (음식 합산의 파생값) |
| 최대 표시 수치 | 99,999 — 초과 시 **99,999 고정** |
| 스크롤 | 스크롤 시 2페이지로 부가 영양소(당류/나트륨/식이섬유/콜레스테롤) 표시 |
| 부가 영양소 갱신 | 음식 수량 or 영양소 변동 시 업데이트 |

### 2.3 식단 수량 수정

| 항목 | 정책 |
|------|------|
| 용도 | 한 끼니 전체의 섭취 비율 조정 (예: 0.5인분) |
| 인터랙션 | 인풋 터치 → 숫자 키보드 출력 (숫자 + 소수점) |
| -/+ 버튼 | 0.1~1: 0.1씩 증감 / 1 이상: 1씩 증감 |
| 입력 정책 | 소수점 둘째 자리까지 입력 가능, 0 입력 불가, 최대 4자리 (예: 99.99) |
| 빈값 해제 시 | 기존 값으로 리셋 |

**계산 로직:**

```
Step 1: 식단 내 모든 음식의 수량 "1" 기준 영양소 합산
Step 2: (1 기준 칼로리) × (입력한 수량) → 각 영양소 동일
Step 3: 칼로리 → 소수점 첫째 자리 반올림 (정수)
        탄/단/지 → 소수점 둘째 자리 반올림 (소수 한 자리)
```

### 2.4 음식 리스트

| 항목 | 정책 |
|------|------|
| 표시 | 음식명 / 중량 / 칼로리 |
| 정렬 | **고칼로리순** |
| **내 음식 뱃지** | 내 음식 등록된 항목에 **뱃지/아이콘** 표시 |
| 터치 | 각 리스트 터치 → 음식 수정 페이지 이동 |
| 최대 개수 | 20개 — 20개 도달 시 추가 버튼 비노출 |

### 2.5 음식 삭제

| 단계 | 동작 |
|------|------|
| 1 | 삭제 버튼(X) 터치 → **"음식 삭제"** 버튼으로 전환 |
| 2 | 전환 후 이외 영역 터치 or **3초간 터치 미발생** 시 → 기존 버튼으로 원복 |
| 3 | "음식 삭제" 터치 → 해당 음식 삭제 |
| 제한 | 음식 리스트 **1개**일 경우 삭제 버튼 **비노출** |
| 부수 효과 | 삭제에 따른 식단명 변동 + 사진 라벨 삭제 + **총 칼로리/탄단지 갱신** |

### 2.6 음식 추가

| 항목 | 정책 |
|------|------|
| 추가 버튼 | 터치 시 음식 검색 페이지 이동 (식단 등록 - 음식 검색과 동일 로직) |
| 추가 시 | 해당 식단 페이지에 음식 추가 |
| **중복 음식 추가** | 이미 푸드렌즈 검색 API 통해 추가한 동일 음식 재추가 시 → 기존 동일 음식 수량에 **+1** + 기존 영양정보 수정사항 초기화(수정 전 값으로) |
| 최대 20개 | 음식 리스트 20개인 경우 추가 버튼 비노출 |

## 3. 음식 수정 페이지

음식 리스트 터치 시 이동하는 개별 음식 수정 화면.

### 3.1 화면 구성

| 영역 | 표시 내용 |
|------|----------|
| 상단 | 음식 이름 |
| 요약 | 칼로리 / 총 내용량 |
| 차트 | 탄/단/지 비율 그래프 및 수치 |
| 영양정보 | 탄/단/지 + 부가영양소 포함 **총 9항목** |

### 3.2 음식 수량 수정

두 가지 입력 타입 지원 (디폴트: 수량).

#### 3.2.1 수량 타입 (디폴트)

| 항목 | 정책 |
|------|------|
| 소수점 | 둘째 자리까지 입력 가능 |
| 0 입력 | **불가** |
| 최대 자릿수 | 4자리 (예: 9999, 99.99) |
| 빈값 해제 | 기존 값으로 리셋 |
| -/+ 버튼 | 수량: 0~1까지 0.1씩, 1 이상 1씩 증감 |
| 0→1 전환 | 0 이하 소수점에서 1 이상 구간으로 증가 시 1로 전환 |

#### 3.2.2 비율 타입 (듀얼 픽커)

| 항목 | 정책 |
|------|------|
| 정수 픽커 | 0 ~ 99 (증감 단위: 1) |
| 분수 픽커 | 0(없음), 1/8, 1/4, 3/8, 1/2, 5/8, 3/4, 7/8 |
| 조합 | 정수 + 분수 (예: 1 + 1/2 = 1.5인분) |
| -/+ 버튼 | 1/8씩 증감 |

#### 3.2.3 타입 전환 변환 테이블

**비율 → 수량:**

| 분수 | 소수 변환 |
|------|----------|
| 1/8 | 0.13 |
| 1/4 | 0.25 |
| 3/8 | 0.38 |
| 1/2 | 0.50 |
| 5/8 | 0.63 |
| 3/4 | 0.75 |
| 7/8 | 0.88 |

**수량 → 비율:**

| 소수 범위 | 분수 변환 |
|----------|----------|
| .00 ~ .06 | 0 (분수 없음) |
| .07 ~ .18 | 1/8 |
| .19 ~ .31 | 1/4 |
| .32 ~ .43 | 3/8 |
| .44 ~ .56 | 1/2 |
| .57 ~ .68 | 5/8 |
| .69 ~ .81 | 3/4 |
| .82 ~ .93 | 7/8 |
| .94 ~ .99 | 정수+1, 분수 0 (반올림) |

#### 3.2.4 키보드 출력 구분

| 인풋 상태 | 키보드 |
|----------|--------|
| 수량으로 입력된 인풋 터치 | 수량 설정 키보드 |
| 비율로 입력된 인풋 터치 | 비율 설정 키보드 (듀얼 픽커) |

#### 3.2.5 수량 변경 계산 로직

```
(음식 1 수량 기준 칼로리) × (입력한 수량) → 새 칼로리
(음식 1 수량 기준 탄수화물) × (입력한 수량) → 새 탄수화물
(음식 1 수량 기준 단백질) × (입력한 수량) → 새 단백질
(음식 1 수량 기준 지방) × (입력한 수량) → 새 지방

반올림: 칼로리 → 정수 / 탄단지 → 소수 한 자리
```

수량 변경 후 **완료 버튼 터치 or 키보드 종료 시** 영양 정보 변동 반영.

### 3.3 칼로리 직접 수정

| 항목 | 정책 |
|------|------|
| 키보드 | 숫자만 입력 가능 |
| 입력 | 정수, 0 가능, 최대 4자리 (9999) |
| 빈값 해제 | 기존 값으로 리셋 |
| 반영 시점 | 포커스 해제 시 |

**계산 로직 (탄/단/지 자동 갱신):**

```
배수 = 수정 칼로리 / 원본 1.0 기준 칼로리
  (배수: 내부적으로 소수점 4자리까지 사용)
  
새 영양소 = 원본 1.0 기준 영양소 × 배수
  (소수점 둘째 자리에서 반올림 → 소수 한 자리)

예외: 기존 탄/단/지 합계가 0이거나 null → 비율 계산 X, 칼로리만 단독 변경
```

### 3.4 탄/단/지 직접 수정

| 항목 | 정책 |
|------|------|
| 키보드 | 숫자 + 소수점 입력 |
| 입력 | 소수점 첫째 자리까지, 0 가능, 최대 4자리 (999.9) |
| 빈값 해제 | 기존 값으로 리셋 |
| 반영 시점 | 포커스 해제 시 |

**계산 로직 (칼로리 자동 갱신):**

```
Step 1: 변동분(Diff) = 입력한 영양소 값 - 수정 전 영양소 값
Step 2: 칼로리 환산
  - 탄수화물/단백질 변동분 × 4 kcal
  - 지방 변동분 × 9 kcal
Step 3: 최종 칼로리 = 기존 칼로리 + Step 2 환산값
  - 소수점 첫째 자리에서 반올림 (정수)
  - 단일 반올림: 최종 계산에서만 단 한 번의 반올림 수행

제한: 계산으로 9999 초과 칼로리 발생 시 최대 표시 수치 9999 제한
```

### 3.5 부가 영양소 수정

| 항목 | 정책 |
|------|------|
| 대상 | 당류 / 나트륨 / 식이섬유 / 트랜스지방 / 포화지방 / 콜레스테롤 |
| 키보드 | 숫자 + 소수점 입력 |
| 입력 | 소수점 첫째 자리까지, 0 가능, 최대 4자리 (999.9) |
| 빈값 해제 | 기존 값으로 리셋 |
| **칼로리 연동** | **없음** — 부가영양소는 수정에 따른 칼로리 변동 없음 (독립적) |

### 3.6 초기화 / 수정완료

| 버튼 | 동작 |
|------|------|
| **초기화** | 모든 인풋 항목 **직전 값**으로 초기화 |
| **수정완료** | 식단 상세페이지로 전환 + 데이터 변동 반영 (필요 시 로딩 로티) |

| 활성화 → 터치 | 비활성화 + 내 음식 리스트 제거 | "내 음식에서 삭제되었습니다." (2.5초) |

**등록 소스별 지원:**

| 음식 소스 | 내 음식 등록/해제 |

| 푸드렌즈 분석 (촬영/앨범) | 지원 |

## 7. 수정 계산 로직 요약

### 7.1 4가지 수정 케이스

| 케이스 | 입력 | 연동 계산 | 반올림 |
|--------|------|----------|--------|
| **식단 수량** | 소수 2자리, 0 불가, 4자리 | (1기준 영양소) × 수량 | 칼로리: 정수 / 탄단지: 소수1 |
| **음식 수량** | 소수 2자리 or 분수, 0 불가 | (1기준 영양소) × 수량 | 칼로리: 정수 / 탄단지: 소수1 |
| **칼로리 수정** | 정수, 0 가능, 4자리 | 배수(소수4자리) × 원본 영양소 | 탄단지: 소수1 |
| **탄단지 수정** | 소수 1자리, 0 가능, 4자리 | 변동분 × 가중치(4/4/9) → 칼로리 가감 | 칼로리: 정수 |

### 7.2 반올림 원칙

| 항목 | 반올림 |
|------|--------|
| 칼로리 | 소수점 첫째 자리에서 반올림 → **정수** |
| 탄/단/지 | 소수점 둘째 자리에서 반올림 → **소수 한 자리** |
| 배수 (칼로리→탄단지) | 내부적으로 소수점 **4자리**까지 사용 |
| **단일 반올림 원칙** | 최종 산출물에서만 단 한 번의 반올림 수행 |

### 7.3 입력 정책 요약

| 대상 | 소수점 | 0 입력 | 최대 자릿수 | 빈값 해제 |
|------|--------|--------|-----------|----------|
| 식단 수량 | 둘째 자리 | 불가 | 4자리 | 기존값 리셋 |
| 음식 수량 (수량형) | 둘째 자리 | 불가 | 4자리 | 기존값 리셋 |
| 음식 수량 (비율형) | 분수(1/8 단위) | — | 99 + 7/8 | — |
| 칼로리 | 정수만 | 가능 | 4자리 | 기존값 리셋 |
| 탄/단/지 | 첫째 자리 | 가능 | 4자리 | 기존값 리셋 |
| 부가영양소 | 첫째 자리 | 가능 | 4자리 | 기존값 리셋 |

## 9. API 기반 플로우

### 9.1 기존 식단 수정 진입

```
[홈 식단 리스트 → 항목 터치]
  │
  ▼
① POST /api/v1/food/diet/sel
   Request: { dietHdNo: {식단 번호} }
   Response: { dietDtCnt, dietHdNm, dietHdImgNm, dietHdImgUrl,
              dietHdNo, dietInsDate,
              foods: [{ foodName, srvSize, srvUnit, srvCnt,
                        nutrition, dietDtNo, dietHdNo, myFoodNo,
                        rsltNo, foodLensId, regSlct, position,
                        srvCntType }] }
  │
  └─ 식단 상세페이지 (기존 데이터 표시, 수정 모드)
```

### 9.3 식단 저장 (등록/수정)

```
[완료 버튼 터치]
  │
  ▼
④ POST /api/v1/food/upsert/check
   Response: { code: "1" (등록 가능) / "-1" (등록 초과) / "0" (에러) }
  │
  ├─ code "1" (등록 가능)
  │   ▼
  │  ⑤ POST /api/v1/food/upsert (Multipart)
  │     Request: { dietHdNo: 0 (신규) 또는 기존값 (수정),
  │               dietHdImgNm,
  │               dietInsDate: "YYYY-MM-DD HH:mm:ss",
  │               foods: [{ foodName, srvSize, srvUnit, srvCnt,
  │                         nutrition, rsltNo, myFoodNo, dietDtNo,
  │                         regSlct, position, srvCntType,
  │                         widgetYn, foodLensId }] }
  │     Response: { code: "1" (성공) / "-1" (실패) / "0" (에러) }
  │  │
  │  └─ 성공 → 홈 복귀 (식단 리스트 + 캘린더 dot + 대시보드 갱신)
  │
  └─ code "-1" (등록 초과)
      → 에러 메시지 표시
```

### 9.5 내 음식 토글

```
[음식 수정 페이지 — 내 음식 토글]
  │
  ├─ 비활성 → 활성 (등록)
  │   ▼
  │  ⑦ POST /api/v1/myFood/upsert
  │     Request: { foodName, srvSize, srvUnit, srvCnt, nutrition,
  │               myFoodNo: 0, srchSlct, rsltNo, dietDtNo, foodLensId }
  │     Response: { result: { myFoodNo } }
  │     → 토스트 "내 음식에 저장되었습니다!" (2.5초)
  │
  └─ 활성 → 비활성 (삭제)
      ▼
     ⑧ POST /api/v1/myFood/del
        Request: { myFoodNo: {삭제할 번호} }
        Response: { code: "1" }
        → 토스트 "내 음식에서 삭제되었습니다." (2.5초)
```

### 9.6 API 엔드포인트 요약

| 순서 | API | Method | Request 주요 필드 | Response | 용도 |
|------|-----|--------|------------------|----------|------|
| ① | /api/v1/food/diet/sel | POST | dietHdNo | FoodDietSelResponse | 기존 식단 조회 |
| ② | /api/v1/lens/detail | POST | code, foodName | FoodSearchResponse | 음식 상세 조회 (검색 추가) |
| ③ | /api/v1/myFood/sel | POST | myFoodNo | FoodSearchResponse | 내 음식 개별 조회 |
| ④ | /api/v1/food/upsert/check | POST | — | code "1"/"-1"/"0" | 등록 가능 여부 확인 |
| ⑤ | /api/v1/food/upsert | POST (Multipart) | dietHdNo, foods, dietInsDate 등 | code "1"/"-1"/"0" | 식단 등록/수정 |
| ⑥ | /api/v1/food/head/del | POST | dietHdNo | code "1"/"0" | 식단 삭제 |

| ⑧ | /api/v1/myFood/del | POST | myFoodNo | code "1" | 내 음식 삭제 |
