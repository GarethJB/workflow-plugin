# 스플래시 / 로그인 기획서

> 작성일: 2026-04-03

---

## 1. 개요

앱 실행 시 서버 선택 → 자동 로그인 판별 → 소셜 로그인 → 가입 여부 분기까지의 진입 플로우를 정의한다.

**시작 조건:** 앱 실행
**완료 조건:** 메인 화면 진입 또는 온보딩 화면 진입

---

## 2. 전체 플로우 차트

```
[앱 실행]
  │
  ▼
[스플래시 화면 진입]
  │
  ├─ 저장된 API URL 조회
  └─ 서버 목록 조회 (GET /api/v1/staff/list)
  │
  ▼
┌──────────────────────────────┐
│ 저장된 API URL 존재?          │
├──────────────────────────────┤
│ Yes → 자동 서버 선택           │
│ No  → 서버 목록 UI 표시        │
│       → 사용자가 서버 선택      │
└──────────────────────────────┘
  │
  ▼
[API URL 설정]
  │
  ▼
┌──────────────────────────────┐
│ 로컬 저장 쿠키 존재?           │
├──────────────────────────────┤
│ Yes → [메인 화면 이동]         │
│       (유효성 검증 없이 즉시)    │
│ No  → [로그인 화면]            │
└──────────────────────────────┘
  │ (쿠키 없음)
  ▼
[로그인 화면]
  │
  ▼
[소셜 로그인 버튼 클릭]
  │ (Naver / Kakao / Google / Apple)
  ▼
[소셜 SDK 인증]
  │
  ├─ 실패 → 에러 메시지 → 버튼 재활성화
  │
  └─ 성공 → 소셜 토큰 획득 (socialSlct + socialKey)
            │
            ▼
      POST /api/v1/auth/isRegistered
      Request: { socialSlct, socialKey }
            │
            ├─ code "-1" (기존 회원)
            │   → POST /api/v1/auth/login
            │     Request: { socialSlct, socialKey }
            │   → 쿠키 로컬 저장 (JSESSIONID, remember-me)
            │   → [메인 화면 이동]
            │
            └─ code "1" (미등록)
                → [온보딩 화면 이동]
```

## 3. 스플래시 화면

### 3.1 화면 구성

| 요소 | 설명 |
|------|------|
| 서버 목록 리스트 | 서버 이름 + API URL 표시 |
| 로딩 인디케이터 | 서버 목록 로딩 중 표시 |
| 에러 메시지 | 서버 목록 조회 실패 시 표시 |

### 3.2 상태별 UI

| 상태 | UI 표시 |
|------|---------|
| Loading | 서버 목록 로딩 중 |
| Content | 서버 목록 리스트 표시 |
| Error | 에러 메시지 표시 |

### 3.3 비즈니스 로직

**초기화:**
1. 저장된 API URL 조회
2. 서버 목록 조회 (`GET /api/v1/staff/list`)
3. 저장된 URL이 존재하면 → 자동 서버 선택
4. 저장된 URL이 없으면 → 서버 목록 UI 표시

**서버 선택:**

| 구분 | 자동 선택 (저장된 URL) | 수동 선택 (사용자 탭) |
|------|----------------------|---------------------|
| 쿠키 초기화 | 하지 않음 | 수행 (저장된 쿠키 삭제) |
| 이후 동작 | API URL 설정 → 쿠키 존재 시 메인 이동 | 쿠키 삭제 → 로그인 화면 이동 |
| 중복 방지 | 3초 내 중복 요청 차단 | 3초 내 중복 요청 차단 |

**자동 로그인 (자동 선택 시):**
- 로컬에 저장된 쿠키가 존재하면 → **유효성 검증 없이 즉시 메인 화면 이동**
  - 쿠키 만료 여부는 메인 화면에서 API 요청 시 서버 응답(401 등)으로 판별
  - 만료 감지 시 메인 화면에서 쿠키 삭제 → 로그인 화면으로 이동
- 저장된 쿠키 없음 → 로그인 화면 이동

## 4. 로그인 화면

### 4.1 화면 구성

| 요소 | 설명 |
|------|------|
| 네이버 로그인 버튼 | 녹색 배경, 네이버 아이콘 |
| 카카오 로그인 버튼 | 노란색 배경, 카카오 아이콘 |
| 구글 로그인 버튼 | 흰색 배경, 구글 아이콘 |
| 애플 로그인 버튼 | 검은색 배경, 애플 아이콘 |
| 로딩 인디케이터 | 로그인 처리 중 표시 |
| 에러 메시지 | 하단 텍스트 |

### 4.2 상태별 UI 동작

| 상태 | 버튼 | 로딩 | 에러 |
|------|------|------|------|
| 초기 | 활성화 | 숨김 | 숨김 |
| 로딩 중 | 비활성화 | 표시 | 숨김 |
| 에러 | 활성화 | 숨김 | 표시 |

### 4.3 소셜 로그인 처리 플로우

```
[소셜 로그인 버튼 클릭]
  │
  ▼
[로딩 표시 + 버튼 비활성화]
  │
  ▼
[소셜 SDK 인증 호출]
  │
  ├─ 실패 → 에러 메시지 표시 → 버튼 재활성화
  │
  └─ 성공 → 소셜 토큰 획득 (socialSlct, socialKey)
            │
            ▼
      [소셜 토큰 로컬 저장]
            │
            ▼
      POST /api/v1/auth/isRegistered
      Request: { socialSlct, socialKey }
            │
            ▼
      ┌──────────────────────────┐
      │ 가입 여부 (code)          │
      ├──────────────────────────┤
      │ "-1" (기존 회원)          │
      │  → POST /api/v1/auth/login │
      │    Request: { socialSlct, socialKey } │
      │  → 쿠키 로컬 저장          │
      │  → 메인 화면 이동          │
      ├──────────────────────────┤
      │ "1" (미등록)              │
      │  → 온보딩 화면 이동         │
      └──────────────────────────┘
```

### 4.4 중복 클릭 방지

- 로딩 중 모든 소셜 로그인 버튼 비활성화
- 연속 탭으로 인한 중복 인증 요청 방지

## 5. API 기반 플로우

### 5.1 스플래시 API 시퀀스

```
[앱 실행]
  │
  ▼
① GET /api/v1/staff/list
   Request: 없음
   Response: [{ serverName, apiUrl, imgServerUrl, photoUrl, photoUploadUrl }]
   실패 → 에러 메시지 표시
  │ 성공
  ▼
[서버 선택 → API URL 설정]
  │
  ▼
┌─────────────────────────────┐
│ 로컬 저장 쿠키 존재?          │
├─────────────────────────────┤
│ Yes → [메인 화면 이동]        │
│       (유효성 검증 없이 즉시)   │
│ No  → [로그인 화면]           │
└─────────────────────────────┘
```

### 5.2 로그인 API 시퀀스

```
[소셜 SDK 인증 성공 → socialSlct + socialKey 획득]
  │
  ▼
② POST /api/v1/auth/isRegistered
   Request: { socialSlct, socialKey }
   Response: code "-1" (기존 회원) / "1" (미등록)
   실패 → 에러 메시지 → 버튼 재활성화
  │
  ├─ code "-1" (기존 회원)
  │   ▼
  │  ③ POST /api/v1/auth/login
  │     Request: { socialSlct, socialKey }
  │     Response: Set-Cookie (JSESSIONID, remember-me)
  │     → 쿠키를 로컬 저장소에 영속 저장
  │     실패 → 에러 메시지 → 버튼 재활성화
  │     성공 → [메인 화면 이동]
  │
  └─ code "1" (미등록)
      → socialSlct, socialKey를 온보딩으로 전달
      → [온보딩 화면 이동]
```

### 5.3 API 엔드포인트 요약

| 순서 | API | Method | Request | Response | 호출 시점 |
|------|-----|--------|---------|----------|----------|
| ① | /api/v1/staff/list | GET | — | 서버 목록 (serverName, apiUrl, imgServerUrl, photoUrl, photoUploadUrl) | 스플래시 초기화 |
| ② | /api/v1/auth/isRegistered | POST | socialSlct, socialKey | code "-1"(기존) / "1"(미등록) | 소셜 인증 성공 후 |
| ③ | /api/v1/auth/login | POST | socialSlct, socialKey | Set-Cookie (JSESSIONID, remember-me) | 기존 회원 확인 후 |

### 5.4 쿠키 처리 정책

| 항목 | 정책 |
|------|------|
| 저장 시점 | `/auth/login` 응답 및 이후 모든 API 응답의 `Set-Cookie` 헤더에서 추출 |
| 저장 위치 | 로컬 저장소 (영속) |
| 갱신 | `Set-Cookie`가 수신될 때마다 저장된 값을 덮어씀 |
| 전송 | 모든 API 요청 시 `Cookie` 헤더에 자동 포함 |
| 자동 로그인 | 앱 재시작 시 저장된 쿠키 존재 여부만 확인 → 존재하면 즉시 메인 이동 (유효성 검증 없음) |
| 만료 감지 | 메인 화면에서 API 요청 시 서버 응답(401 등)으로 판별 → 쿠키 삭제 → 로그인 화면 이동 |
| 삭제 시점 | 수동 서버 변경 시 / 메인에서 세션 만료 감지 시 |

## 6. 화면 전이 매트릭스

| 현재 화면 | 이동 대상 | 조건 |
|----------|----------|------|
| 스플래시 | 메인 화면 | 저장된 쿠키 존재 (유효성 검증 없이 즉시) |
| 스플래시 | 로그인 화면 | 저장된 쿠키 없음 / 수동 서버 선택 |
| 로그인 | 메인 화면 | 소셜 로그인 성공 + 기존 회원 (code "-1") |
| 로그인 | 온보딩 화면 | 소셜 로그인 성공 + 미등록 (code "1") |

## 7. 에러 처리

| 에러 상황 | 처리 |
|----------|------|
| 서버 목록 조회 실패 | 에러 메시지 표시 |
| 소셜 SDK 인증 실패 | 에러 메시지 표시 → 버튼 재활성화 |
| isRegistered API 실패 | 에러 메시지 표시 → 버튼 재활성화 |
| login API 실패 | 에러 메시지 표시 → 버튼 재활성화 |
| login API code "-11" (계정 없음) | 에러 메시지 표시 → 버튼 재활성화 |
| 세션 만료 (메인에서 401 감지) | 저장된 쿠키 삭제 → 로그인 화면 이동 |
| 네트워크 오류 | 에러 메시지 표시 → 버튼 재활성화 |
