## 1. MVP 전체 플로우차트

### 1.1 앱 시작 ~ 메인 진입

```
[앱 실행]
    │
    ▼
[스플래시 화면]
    │
    ├─ 저장된 API URL 조회
    └─ GET /api/v1/staff/list (서버 목록 조회)
    │
    ▼
┌──────────────────────┐
│ 저장된 API URL 존재?   │
├──────────────────────┤
│ Yes → 자동 서버 선택    │
│ No  → 서버 목록 UI     │
│       → 사용자 서버 선택 │
└──────────────────────┘
    │
    ▼
[API URL 설정]
    │
    ▼
┌──────────────────────┐
│ 로컬 저장 쿠키 존재?   │
├──────────────────────┤
│ Yes → [홈] (유효성 검증 │
│       없이 즉시 이동)   │
│ No  → [로그인 화면]    │
└──────────────────────┘
    │ (쿠키 없음)
    ▼
[로그인 화면 — 소셜 로그인]
    │ (Naver / Kakao / Google / Apple)
    ▼
[소셜 SDK 인증 → socialSlct + socialKey 획득]
    │
    ▼
POST /api/v1/auth/isRegistered
    │
    ├─ code "-1" (기존 회원)
    │   → POST /api/v1/auth/login { socialSlct, socialKey }
    │   → 쿠키 로컬 저장 (JSESSIONID, remember-me)
    │   → [홈 (메인페이지)]
    │
    └─ code "1" (미등록)
        → [온보딩 5단계]
            │
            ├─ Step 1: 성별 선택 (즉시 이동)
            ├─ Step 2: 출생연도 입력
            ├─ Step 3: 신장·체중 입력
            ├─ Step 4: 활동량 선택
            ├─ Step 5: 목표 선택
            │
            ▼
      [TDEE 결과 화면]
            │
      (시작하기 버튼)
            │
            ├─ ① 영양소 계산 API (nutrition/calculate)
            ├─ ② 회원 등록 API (member/register + socialSlct, socialKey)
            ├─ ③ 로그인 API (auth/login + socialSlct, socialKey)
            └─ ④ 영양소 등록 API (nutrition/register)
                  │
                  ▼
           [홈 (메인페이지)]
```

**스플래시/로그인 정책:**
- 자동 로그인: 로컬 쿠키 존재 여부만 확인 → 존재 시 유효성 검증 없이 즉시 메인 이동
- 세션 만료: 메인에서 API 요청 시 401 응답으로 감지 → 쿠키 삭제 → 로그인 화면 이동
- 수동 서버 선택 시: 저장된 쿠키 삭제 → 로그인 화면으로 이동
- 인증: 소셜 토큰 기반 (socialSlct + socialKey)

**온보딩 정책:**
- 각 단계 뒤로가기 시 이전 입력 데이터 유지
- 앱 종료 후 재진입 시 Step 1부터 재시작 (상태 미저장)
- 시작하기 버튼 API 호출 중 비활성화 (중복 가입 방지)

### 1.2 메인페이지 구조

```
┌──────────────────────────────────────┐
│            홈 (메인페이지)              │
│                                      │
│  ┌─────────────────────────────┐     │
│  │ 캘린더 (좌/우 스크롤)          │     │
│  │ 2025년~ / 미래 터치 불가       │     │
│  └─────────────────────────────┘     │
│                                      │
│  ┌─────────────────────────────┐     │
│  │ 대시보드 (3페이지 스와이프)      │     │
│  │ 1p: 칼로리 반도넛 + 탄단지     │     │
│  │ 2p: 부가 영양소 (막대 그래프)   │     │
│  │ 3p: 음수량 (+/- 0.25L)       │     │
│  └─────────────────────────────┘     │
│                                      │
│  ┌─────────────────────────────┐     │
│  │ 식단 리스트 (최근 등록순)       │     │
│  │ 스와이프 삭제 + 컨펌           │     │
│  └─────────────────────────────┘     │
│                                      │
│                          [FAB] ──→ 식단 등록 │
└──────────────────────────────────────┘
```

**데이터 갱신 시점:**
- 식단 등록/삭제 직후 즉시 반영
- 디바이스 기준 날짜 변동(자정 경과) 시
- 탭 이동 후 복귀 시
- 앱 백그라운드→포그라운드 전환 시

### 1.3 식단 등록 진입점 (FAB)

```
[FAB 터치]
    │
    ├──→ [사진 찍기] ──→ 카메라 권한 체크
    │     │
    │     └──→ 카메라 뷰 (3모드 탭)
    │           ├─ 음식 스캔 (디폴트) → 푸드렌즈 API
    │           ├─ 바코드 → 바코드 DB/API 조회
    │           └─ 라벨 → OCR + LLM 추출
    │
    ├──→ [앨범 선택] ──→ 앨범 권한 체크
    │     │
    │     └──→ 시스템 앨범 피커 → 시간 선택 팝업 (당일만) → 푸드렌즈 API
    │
    └──→ [음식 검색] ──→ 음식 검색 페이지
          │
          ├─ 음식 검색 탭: 텍스트 검색 (0.3초 디바운스)
          └─ 내 음식 탭: 내 음식 목록 (30개 페이징)

    ※ 모든 경로 → [식단 상세페이지]로 수렴
```

### 1.4 식단 상세페이지 공통 플로우

```
[식단 상세페이지] ← 4가지 진입 경로
    │
    ├─ 사진 영역 (사진 없으면 대체 이미지)
    ├─ 총 칼로리/탄단지 요약 (수정 불가, 파생값)
    ├─ 탄/단/지 게이지 비율 연동
    ├─ 부가 영양소 (2p 스크롤)
    │
    ├─ 음식 리스트 (고칼로리순, 최대 20개)
    │   ├─ 항목 터치 → 음식 수정 페이지
    │   ├─ X 버튼 → "음식 삭제" 전환 → 삭제 (3초 타임아웃)
    │   └─ 내 음식 뱃지 표시
    │
    ├─ 음식 추가 → 음식 검색 페이지
    ├─ 수량 수정 → 영양정보 재계산
    │
    ├─ [완료] → 등록 가능 확인 (upsert/check) → 식단 DB 저장 → 홈 이동 + 리스트/캘린더/대시보드 갱신
    ├─ [뒤로가기 (미저장)] → 작성 취소 컨펌
    └─ [삭제 (기록 완료 후)] → 컨펌 → 삭제 API → 홈 이동
```

### 1.5 음식 수정 플로우

```
[음식 수정 페이지]
    │
    ├─ 음식명 / 칼로리 / 총 내용량
    ├─ 탄/단/지 비율 그래프 + 수치
    ├─ 영양정보 총 9항목
    │
    ├─ 수량 수정 (수량형 / 비율형 듀얼 픽커)
    │   └─ 타입 전환 시 값 변환 (1/8 단위 ↔ 소수)
    │
    ├─ 칼로리 수정 → 탄단지 자동 갱신 (배수 × 원본, 소수4자리)
    ├─ 탄단지 수정 → 칼로리 자동 가감 (변동분 × 4/4/9 kcal)
    ├─ 부가 영양소 수정 → 칼로리 연동 없음 (독립)
    │
    ├─ 내 음식 등록 토글 (토스트 2.5초 + 작동 방지)
    ├─ [초기화] → 직전 값 복원
    └─ [수정완료] → 식단 상세페이지 복귀 + 데이터 반영
```

### 2.2 온보딩 (Onboarding)

| 단계 | 화면 상태 | 사용자 행동 | 시스템 처리 |
|------|----------|-----------|-----------|
| 1 (1/5) | 성별 선택 | 남성/여성 터치 | **즉시** 다음 페이지 이동 (다음 버튼 없음) |
| 2 (2/5) | 출생연도 입력 | 휠 픽커 조절 | 범위: 1900~2026, 디폴트 1991, 경계값 바운스 없이 정지 |
| 3 (3/5) | 신장·체중 입력 | 듀얼 휠 픽커 | 신장 60~220cm, 체중 30~300kg, 성별별 디폴트, 햅틱 |
| 4 (4/5) | 활동량 선택 | 4개 항목 중 선택 | 단일 선택, 재터치 해제, 선택 전 다음 버튼 비활성화 |
| 5 (5/5) | 목표 선택 | 감량/유지/증량 선택 | 단일 선택, 재터치 해제, 선택 전 다음 버튼 비활성화 |
| 6 | TDEE 로딩 | — | "입력한 정보로 목표값을 찾는 중입니다..." |
| 7 | TDEE 결과 | 반도넛 차트 + 수치 확인 | 결과값 정수 반올림 표시, 반도넛 좌→우 애니메이션 |
| 8 | 시작하기 | 시작하기 터치 | ① nutrition/calculate → ② member/register(+socialSlct, socialKey) → ③ auth/login(+socialSlct, socialKey) → ④ nutrition/register → 홈 |

**비즈니스 규칙:**
- 영양소 계산은 서버 API에서 처리 (POST /api/v1/nutrition/calculate)
- 회원 등록/로그인 시 소셜 토큰 (socialSlct + socialKey) 필수

**숫자 표기:** 모든 수치 정수 (반올림), 천 단위 콤마, 섭취량 최대 99,999 고정, 권장량 최대 9,999 고정

### 2.4 음식 스캔 (ScanFood)

3가지 촬영 모드 + 앨범 선택으로 구분. 모든 경로 → 식단 상세페이지.

#### 2.4.1 음식 스캔 (푸드렌즈) 플로우

| 단계 | 화면 상태 | 사용자 행동 | 시스템 처리 |
|------|----------|-----------|-----------|
| 1 | 카메라 뷰 (음식 스캔 디폴트) | 식단 등록 진입 | 카메라 권한 체크 (거절 시 설정 유도 팝업 → 앱 설정 이동) |
| 2 | 촬영 | 셔터 터치 | 사진 캡처 → 푸드렌즈 API 전송 → 로딩 로티 |
| 3 | 식단 상세 | — | 촬영 사진 + 분석 데이터 반영 (음식명 + 영양정보 자동입력) |
| 실패 | — | — | 식단 이름 → "음식이 아님", 모든 영양소 0으로 진행 |

#### 2.4.2 바코드 스캔 플로우

| 단계 | 화면 상태 | 사용자 행동 | 시스템 처리 |
|------|----------|-----------|-----------|
| 1 | 카메라 뷰 (바코드 탭) | 바코드에 카메라 조준 | 바코드 인식 → 즉시 DB/API 조회 |
| 2a | 제품 있음 | — | 식단 상세페이지 이동 + 영양정보 반영 |
| 2b | 제품 없음 | — | OCR 유도 팝업: "영양성분표를 사진으로 찍으면 자동으로 읽어올 수 있어요!" |
| 2b+ | 유도 팝업 | 스캔하기 터치 | 라벨 모드로 전환 |
| 실패 | — | — | 카메라 뷰 재전환 + 재시도 문구 (2.5초 후 자동 숨김) |

#### 2.4.3 영양성분 OCR 플로우

| 단계 | 화면 상태 | 사용자 행동 | 시스템 처리 |
|------|----------|-----------|-----------|
| 1 | 카메라 뷰 (라벨 탭) | 영양성분표 촬영 | OCR + LLM 데이터 맵핑 → 로딩 로티 |
| 2 | 분석 결과 확인 | 음식명 입력 + 영양정보 확인/수정 | 헤더 "영양정보 분석 결과" |
| 3 | 저장 | 저장 버튼 (음식명 입력 시 활성화) | 식단 상세페이지 이동 + 데이터 반영 |
| 실패 | — | — | 카메라 뷰 재전환 + 재시도 문구 (2.5초 후 자동 숨김) |

**OCR 분석 결과 입력 정책:**
- 음식명: 필수, 1~20자, 특수문자 가능, 이모지 불가, 공백 정규화
- 영양정보: 숫자 키보드, 소수점 1자리, 칼로리/내용량은 정수만, 최대 4자리, 빈값 시 리셋

#### 2.4.4 앨범 선택 플로우

| 단계 | 화면 상태 | 사용자 행동 | 시스템 처리 |
|------|----------|-----------|-----------|
| 1 | 앨범 피커 | 사진 선택 (단일) | 앨범 권한 체크 (거절 시 설정 유도 팝업 → 앱 설정 이동) |
| 2 | 시간 선택 팝업 (당일만) | 촬영시간/현재시간 선택 | 당일 외 → 팝업 X, 오후 11:59 기준 자동 등록 |
| 3 | 분석 중 | — | 푸드렌즈 API → 로딩 로티 |
| 4 | 식단 상세 | — | 촬영 사진 + 분석 데이터 반영 |

### 2.5 음식 검색 (FoodSearch)

| 단계 | 화면 상태 | 사용자 행동 | 시스템 처리 |
|------|----------|-----------|-----------|
| 1 | 검색 화면 | 진입 | 인풋 자동 포커스 + 키보드 실행, 최근 검색 칩 표시 (최대 10개) |
| 2 | 검색 결과 | 음식명 입력 | 0.3초 디바운스 → 검색 API (공백 제거, 1글자 이상) |
| 3 | 음식 선택 | 결과 항목 터치 | 식단 상세페이지 이동 (음식 추가 + 식단 이름 자동 입력) |
| alt | 내 음식 탭 | 탭 전환 | 내 음식 리스트 (최근 등록순, 30개 페이징), 인풋 값 탭별 독립 관리 |
| alt | 없음 | — | "요청하신 음식을 찾을 수가 없어요" / "등록 된 내 음식이 없어요" |

### 2.6 식단 상세 (FoodDetail)

| 단계 | 화면 상태 | 사용자 행동 | 시스템 처리 |
|------|----------|-----------|-----------|
| 1 | 식단 상세 | 진입 | 4가지 경로별 데이터 로드, 사진 없으면 대체 이미지 |
| 2 | 음식 목록 | — | 고칼로리순 정렬, 음식명/중량/칼로리, 내 음식 뱃지 |
| 3a | 수량 수정 | 수량 인풋/스테퍼 | (1기준 영양소) × 수량, 칼로리 정수·탄단지 소수1 반올림 |
| 3b | 음식 추가 | 추가 버튼 | 음식 검색 이동, 동일 음식 재추가 시 수량 +1, 최대 20개 |
| 3c | 음식 수정 | 리스트 항목 터치 | 음식 수정 페이지 → 칼로리↔탄단지 양방향 자동 계산 |
| 3d | 음식 삭제 | X → "음식 삭제" (3초) | 삭제 → 식단명 + 영양정보 + 사진 라벨 갱신, 1개 시 비노출 |
| 4 | 저장 | 완료 버튼 | 등록 가능 확인 (upsert/check) → 식단 DB 저장 (Multipart) → 홈 이동 + 리스트/캘린더/대시보드 갱신 |
| alt | 사진 등록 | 사진 영역 터치 | 사진 찍기/앨범 선택 레이어 → 촬영/선택 → 업로드 |
| alt | 내 음식 토글 | 버튼 터치 | 등록/삭제 토스트 2.5초 + 작동 방지, 소스별 지원, 크로스 식단 공유 |
| alt | 삭제 | 더보기 → 삭제 (기록 완료 후) | 컨펌 → 삭제 API → 홈 이동 |
| alt | 뒤로가기 | 뒤로가기 (미저장) | 작성 취소 컨펌 → "작성 취소" 시 홈 복귀 + 식단 소멸 |
| 에러 | 세션 만료 | — | API 응답 401 감지 → 저장된 쿠키 삭제 → 로그인 화면 이동 |

**진입 경로 4가지:**

| 경로 | 출발점 | 데이터 소스 | 모드 |
|------|-------|-----------|------|
| 기존 식단 클릭 | 홈 식단 리스트 | 식단 음식 목록 조회 API | 수정 |

| 검색 결과 | 음식 검색 | 음식 상세 조회 API | 신규 |
| 내 음식 선택 | 내 음식 목록 | 내 음식 개별 조회 API | 신규 |

| **서버 목록 조회** (GET /staff/list) | 스플래시 초기화 | — | 서버 목록 (serverName, apiUrl 등) |

## 5. 핵심 분기 정책

### 5.1 앱 진입 분기

```
[스플래시]
  │
  ▼
로컬 쿠키 존재?
  ├─ Y → 유효성 검증 없이 즉시 홈 이동 (만료 시 메인에서 401로 감지)
  └─ N → 로그인 화면 → 소셜 인증 → isRegistered
            │
            ├─ code "-1" (기존 회원) → auth/login → 쿠키 저장 → 홈
            └─ code "1" (미등록) → 온보딩 → 시작하기 → 가입 + 로그인 + 영양소 등록 → 홈
```

### 5.2 바코드 음식정보 존재 여부

```
바코드 제품 정보?
  ├─ 있음 → 식단 상세페이지 (영양정보 반영)
  └─ 없음 → OCR 유도 팝업 → "스캔하기" → 라벨 모드 전환
```

### 5.3 앨범 시간 선택 팝업

```
캘린더 설정 일자 = 당일?
  ├─ Y → 시간 선택 팝업 (촬영시간 / 현재시간)
  └─ N → 팝업 X, 해당 일자 오후 11:59 기준 자동 등록
```

### 5.4 식단 상세 — 수정 시 데이터 반영

```
음식 정보 변동
  ├─ 수량 변경 → (1기준 영양소) × 수량 → 식단 총 칼로리/탄단지 갱신
  ├─ 칼로리 수정 → 배수 기반 탄단지 갱신 (합계 0이면 단독 변경)
  ├─ 탄단지 수정 → 변동분 × 가중치 → 칼로리 가감
  ├─ 부가 영양소 수정 → 칼로리 변동 없음
  ├─ 음식 추가/삭제 → 식단명 재산정 + 영양정보 갱신 + 사진 라벨 갱신
  └─ 모든 변동 → 즉시 식단에 반영 (파생값)
```

### 5.5 식단 저장 후 홈 반영 범위

```
[완료 버튼] → food/upsert/check (등록 가능 확인) → food/upsert (저장)

식단 등록/수정/삭제 완료
  ├─ 식단 리스트 갱신
  ├─ 캘린더 dot 갱신
  └─ 대시보드 수치 갱신 (칼로리/탄단지/부가영양소)
```
