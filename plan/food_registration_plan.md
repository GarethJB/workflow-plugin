# 식단 등록 / 수정 기획서

> Figma 소스: AI 식단 분석 서비스 MVP / 스토리보드 - 식단 등록 / 수정
> 작성일: 2026-04-03

---

## 1. 개요

홈 화면의 FAB를 통해 식단을 등록하고, 등록된 식단을 수정·삭제한다. 진입 방식은 3가지(사진 찍기 / 앨범 선택 / 음식 검색)이며, 모든 경로는 **식단 상세페이지**로 수렴한다.

---

## 2. 식단 / 음식 정의

| 개념 | 정의 |
|------|------|
| **식단** | 집합 — 아침, 점심 등 끼니를 나타내는 상위 컨테이너 |
| **음식** | 원소 — 식단에 포함된 개별 메뉴(현미밥, 닭가슴살 등). 고유의 영양 데이터를 가짐 |
| **관계** | 식단은 음식들의 합으로 구성. 식단의 영양정보는 **원소의 파생값**이며 독자 존재 불가 |

### 2.1 식단 영양정보 산출

```
식단 영양정보 = 개별 음식의 합 × 식단 전체 섭취 비율
```

### 2.2 데이터 수정 권한

| 대상 | 수정 가능 여부 |
|------|-------------|
| 식단의 칼로리 및 영양정보 | **수정 불가** (음식 파생값) |
| 식단 섭취 비율 | 수정 가능 |
| 개별 음식 | 수정 가능 |
| 음식 추가/삭제/변경 | **즉시 식단에 반영** |

### 2.3 영양소-칼로리 연동 정책

**가중치 상수:**
- 1g 탄수화물 = 4 kcal
- 1g 단백질 = 4 kcal
- 1g 지방 = 9 kcal

| 수정 방향 | 계산 |
|----------|------|
| 탄/단/지 수정 시 | 기존 칼로리에 영양소 변동분(탄·단×4, 지×9)을 가감 |
| 칼로리 수정 시 | 직전 칼로리 대비 증감 비율(New/Old)을 계산, 탄/단/지에 각각 곱하여 갱신 |
| 소수점 | 소수점 첫째자리 반올림 |
| 예외 | 기존 탄/단/지 합계가 0일 경우 갱신하지 않음 |

## 3. 식단 등록 정책

### 3.1 필수값 / 선택값

| 구분 | 항목 | 정책 |
|------|------|------|
| **필수** | 식단 등록 시간·날짜 | YYYY-MM-DD HH:mm |
| **필수** | 음식명 | 푸드렌즈 API 명칭 기준 자동입력. 빈 문자열 불가 |
| **필수** | 기준 내용량 | 푸드렌즈 API 기준 자동입력. OCR/내 음식 등록 시 직접 입력. 음수 불가, 0 가능 |
| **필수** | 칼로리 및 영양소 | 음수 불가, 0 가능 |
| **선택** | 음식 사진 | 유저 업로드 사진. 없을 경우 대체 일러스트 출력 |
| **선택** | 부가 영양소 | 없을 경우 0으로 표시 |

### 3.2 영양성분 정보 구조

**식단 영양성분:**

| 카테고리 | 항목 |
|---------|------|
| 에너지 | 칼로리 |
| 주요 영양소 | 탄수화물 / 단백질 / 지방 |
| 부가 영양소 | 당류 / 나트륨 / 식이섬유 / 포화지방 / 트랜스지방 / 콜레스테롤 |

**음식 영양성분:**

| 카테고리 | 항목 | 연동 |
|---------|------|------|
| 내용량 | 총 내용량 | — |
| 에너지 | 칼로리 | 변동 시 탄/단/지 연계 |
| 주요 영양소 | 탄수화물 / 단백질 / 지방 | 변동 시 칼로리 연계 |
| 부가 영양소 | 당류 / 나트륨 / 식이섬유 / 포화지방 / 트랜스지방 / 콜레스테롤 | — |

### 3.3 등록 시간 정책

| 조건 | 등록 시간 |
|------|----------|
| 캘린더 설정 일자 기준 | 해당 일자 **오후 11:59** 기준으로 등록 |
| 앨범 선택 (당일) | 촬영시간 or 현재시간 중 사용자 선택 |
| 앨범 선택 (당일 외) | 팝업 출력 없음 (오후 11:59 기준) |

## 4. 진입 경로 1 — 사진 찍기

### 4.1 카메라 권한 체크

| 상태 | 동작 |
|------|------|
| 미정 (최초) | 시스템 권한 팝업 노출 |
| 허용 | 즉시 카메라 실행 |
| 거절 | 설정 유도 팝업 → "식단 사진을 촬영하려면 카메라 권한이 필요해요. 설정에서 권한을 허용해 주세요." |
| 거절 팝업 확인 시 | **앱 설정 → 카메라 권한 화면** 이동 |

### 4.2 카메라 촬영 모드

| 항목 | 정책 |
|------|------|
| 촬영 타입 선택 탭 | **바코드** / **음식 스캔** / **라벨** |
| 디폴트 모드 | 음식 스캔 |
| 각 모드별 가이드 | 촬영 가이드 디자인 출력 |

#### 4.2.1 음식 스캔 (푸드렌즈)

| 단계 | 동작 |
|------|------|
| 1 | 촬영 시 **푸드렌즈 API** 전송 |
| 2 | 필요 시 로딩 로티 출력 |
| 3 | 식단 상세페이지 전환 — 직전 촬영 사진 및 분석 데이터 반영 |
| 인식 성공 | 푸드렌즈 API 기준 식단 정보 자동 입력 (음식 이름 + 영양정보) |
| 인식 실패 | 식단 이름 → "음식이 아님", 영양정보 0인 상태로 진행 |

#### 4.2.2 바코드 스캔

| 단계 | 동작 |
|------|------|
| 1 | 바코드 촬영 가이드 디자인 출력 |
| 2 | 바코드 인식 시 **즉시** 바코드 번호 DB or API 조회 |
| 3a | 제품 정보 **있음** → 식단 상세페이지 이동, 바코드 기반 영양정보 데이터 반영 |
| 3b | 제품 정보 **없음** → 영양성분 분석 유도 팝업 출력 |
| 유도 팝업 | "대신 영양성분표를 사진으로 찍으면 영양 정보를 자동으로 읽어올 수 있어요!" |
| 스캔하기 | 터치 시 **라벨 모드**로 전환 |
| **스캔 실패** | 카메라 뷰로 재전환 + 상단 영역 재시도 문구 표시 → **2.5초 후 비노출** |

#### 4.2.3 영양정보 라벨 OCR

| 단계 | 동작 |
|------|------|
| 1 | 영양정보 라벨 촬영 가이드 디자인 출력 |
| 2 | 촬영 시 데이터 맵핑 진행 (필요 시 로딩 로티) |
| 3 | 완료 시 **분석 결과 확인 페이지** 이동 |
| **스캔 실패** | 카메라 뷰로 재전환 + 상단 영역 재시도 문구 표시 → **2.5초 후 비노출** |

**분석 결과 확인 페이지:**

| 항목 | 정책 |
|------|------|
| 헤더 | "영양정보 분석 결과" |
| 음식명 입력 (필수) | 최소 1자 ~ 최대 20자. 특수문자 가능, 이모지 불가 |
| 공백 처리 | 앞뒤 공백 제거, 연속 공백 단일 공백으로 치환, 공백만 입력 시 내용 없음 |
| 영양정보 인풋 | 칼로리 / 탄수화물 / 단백질 / 지방 / 당류 / 나트륨 / 식이섬유 / 콜레스테롤 / 포화지방 |
| 저장 버튼 | 음식명 입력 시 활성화 → 터치 시 식단 상세페이지 이동, 저장된 음식 데이터 반영 |

**영양정보 인풋 입력 정책:**

| 항목 | 정책 |
|------|------|
| 키보드 | 숫자 입력 키보드 |
| 소수점 | 첫째 자리 단위 입력 가능, 0 입력 가능 |
| 칼로리/내용량 | **정수만** 입력 가능 |
| 최대 자릿수 | 4자리 (예: 9999, 999.9) |
| 빈값 포커스 해제 | 기존 값으로 리셋 |

## 5. 진입 경로 2 — 앨범 선택

### 5.1 앨범 권한 체크

| 상태 | 동작 |
|------|------|
| 미정 (최초) | 시스템 권한 팝업 노출 |
| 허용 | 시스템 앨범 피커 실행 |
| 거절 | 설정 유도 팝업 → "식단 사진을 불러오기 위해 앨범 접근 권한이 필요해요. 설정에서 권한을 허용해 주세요." |
| 거절 팝업 확인 시 | **앱 설정 → 앨범 권한 화면** 이동 |

### 5.2 앨범 플로우

| 단계 | 동작 |
|------|------|
| 1 | 시스템 앨범 피커 실행 (사진 중복 선택 불가) |
| 2 | 사진 선택 시 **시간 기준 선택 팝업** 출력 (앨범 레이어 내림 처리 후) |
| 3 | 시간 선택 후 푸드렌즈 API 전송 처리 |
| 4 | 식단 상세페이지 출력 + 분석 데이터 반영 |

**시간 기준 선택 팝업:**

| 항목 | 정책 |
|------|------|
| 문구 | "사진이 촬영된 날짜로 기록할지, 현재 시간 기준으로 기록할지 선택해 주세요." |
| 촬영시간 선택 | 사진 메타데이터 기준 일자/시간 |
| 현재시간 선택 | 현재 디바이스 시간 기준 |
| 출력 조건 | 등록 시점 메인페이지 설정 일자가 **당일인 경우에만** 팝업 출력 (당일 아닌 경우 팝업 X) |

## 6. 진입 경로 3 — 음식 검색

### 6.1 음식 검색 페이지

| 항목 | 정책 |
|------|------|
| 진입 시 | 인풋창 활성화 (시스템 키보드 실행) |
| 등록 시간 | 캘린더 설정 일자 기준, 오후 11:59 |

### 6.2 검색 인풋

| 항목 | 정책 |
|------|------|
| (x) 버튼 | 1글자 이상 입력 시 우측에 텍스트 전체 삭제 버튼 노출 |
| 플레이스홀더 | 비노출 |
| 최대 글자수 | 20자 |
| 키보드 내림 | 검색 리스트 영역 스크롤 시 |
| X 버튼 (우측 상단) | 페이지 닫기 |

### 6.3 최근 검색 음식

| 항목 | 정책 |
|------|------|
| 출력 조건 | 인풋창에 값이 없을 때 출력 / **값 입력 시 비노출** (검색 결과 리스트로 전환) |
| 최대 개수 | 10개 (11번째 검색 시 **가장 오래된 항목 자동 제거**) |
| 정렬 | 최근 선택 순 |
| 표시 형식 | 음식명 칩 디자인 |
| 터치 시 | 해당 검색어 자동 입력 + **검색 API 즉시 호출** |
| x 버튼 | 즉시 삭제 |
| 삭제 | 가능 |

### 6.4 검색 결과 리스트

| 항목 | 정책 |
|------|------|
| API 호출 | 인풋 입력 값 변동 시 **0.3초** 디바운스 |
| 공백 처리 | 앞뒤 공백 제거, 최소 1글자 이상부터 동작 |
| 표시 | 음식명 + 제조사 (제조사 없을 경우 미표시) |
| 음식 없음 | "요청하신 음식을 찾을 수가 없어요" |
| 음식 선택 시 | 식단 상세페이지 이동 (선택한 음식 추가 상태, 식단 이름 자동 입력) |

### 6.5 탭 전환 (음식 검색 ↔ 내 음식)

| 항목 | 정책 |
|------|------|
| 인풋 값 | 각 탭 **별도 관리** (탭 전환 시 기존 값 유지) |
| 예시 | 음식 검색에서 "우유 식빵" 입력 → 내 음식 탭 전환 시 미입력 → 재전환 시 "우유 식빵" 유지 |

### 6.6 내 음식 탭

| 항목 | 정책 |
|------|------|
| 리스트 정렬 | 최근 등록순 |
| 페이징 | 30개 단위 |
| 표시 | 음식명 / 제조사 / 칼로리 (제조사 없을 경우 미표시) |
| 터치 시 | 해당 음식 추가 → 식단 상세페이지 |
| 검색 | 인풋 입력 시 0.3초 디바운스 검색 API 호출 |
| 빈 리스트 | "등록 된 내 음식이 없어요" |
| 검색 결과 없음 | "요청하신 음식을 찾을 수가 없어요" |
| 내 음식 추가 버튼 | 내 음식 등록 프로세스 진행 |

## 7. 내 음식 등록 페이지

| 촬영 시 | 영양성분 추출 프로세스 진행 → 인풋에 자동 반영 |

## 8. 식단 상세페이지 (공통)

모든 진입 경로(사진 찍기/앨범/검색/기존 식단)가 수렴하는 핵심 화면.

### 8.1 음식 추가

| 항목 | 정책 |
|------|------|
| 추가 버튼 | 터치 시 음식 검색 페이지 전환 (인풋창 활성화) |
| 추가 시 | 추가한 음식 영양정보 합산 + 식단 이름 갱신 |
| 음식 터치 | 양 수정 페이지 이동 |

### 8.2 사진 등록 (식단 상세에서)

| 항목 | 정책 |
|------|------|
| 사진 없는 상태 | 사진 영역에 **대체 일러스트** 출력 |

**사진 영역 터치 시 선택 레이어 출력:**

| 옵션 | 동작 |
|------|------|
| 사진 찍기 | 카메라 실행 (권한 체크) → 촬영 → 사진 영역에 업로드 |
| 앨범 선택 | 앨범 실행 (권한 체크) → 선택 → 사진 영역에 업로드 |

사진 업로드 후 **기록 완료** 터치 시 해당 사진 저장 (Multipart 업로드 포함).

### 8.3 뒤로가기 (최초 저장 전)

| 항목 | 정책 |
|------|------|
| 트리거 | 식단 상세페이지 생성 후 뒤로가기 (최초 저장 X) |
| 동작 | **작성 취소 컨펌** 출력 |
| 작성 취소 버튼 | 메인페이지 복귀 + 해당 식단 페이지 소멸 |
| 아니오 버튼 | 컨펌 취소 (식단 상세 유지) |

### 8.4 완료 (저장)

| 항목 | 정책 |
|------|------|
| 동작 | 완료 버튼 터치 시 메인페이지 이동 + **식단 리스트에 해당 식단 표시** |
| 로딩 | 필요 시 로딩 로티 진행 |
| API | POST /api/v1/food/upsert (Multipart — 이미지 있을 경우) |
| 반영 범위 | 홈 식단 리스트 + 캘린더 dot + 대시보드 영양소 수치 갱신 |

## 12. API 기반 플로우

### 12.1 진입 경로별 분석 API

```
[진입 경로 1 — 사진 찍기 (음식 스캔)]
  │
  ▼
① POST /api/v1/lens/analyze (Multipart)
   Request: { file: {촬영 이미지}, srchSlct: "a" }
   Response: { foods: [{ foodName, srvSize, srvUnit, srvCnt,
              nutrition, position, foodLensId, rsltNo }],
              dietHdImgNm, dietHdImgNmUrl }
  │
  ├─ 인식 성공 → 식단 상세페이지 (분석 데이터 반영)
  └─ 인식 실패 → 식단 상세페이지 (음식명 "음식이 아님", 영양정보 0)
```

```
[진입 경로 1 — 사진 찍기 (바코드)]
  │
  ▼
② POST /api/v1/foodsafety/barcode
   Request: { barcode: "{인식된 바코드}" }
   Response: { foods: [...], dietHdImgNm, dietHdImgNmUrl }
  │
  ├─ 제품 정보 있음 → 식단 상세페이지 (바코드 데이터 반영)
  └─ 제품 정보 없음 → OCR 유도 팝업 → 라벨 모드 전환
```

```
[진입 경로 1 — 사진 찍기 (라벨 OCR)]
  │
  ▼
③ POST /api/v1/ocr (Multipart)
   Request: { file: {라벨 이미지} }
   Response: { foods: [{ foodName, srvSize, srvUnit, srvCnt,
              nutrition, rsltNo: -1, myFoodNo: -1 }],
              dietHdImgNm: "", dietHdImgNmUrl: "" }
  │
  ├─ 성공 → OCR 결과 확인 페이지 (음식명 입력 + 영양정보 수정)
  │         → 저장 → 식단 상세페이지
  └─ 실패 → 카메라 뷰 재전환 + 재시도 문구 (2.5초 후 숨김)
```

```
[진입 경로 2 — 앨범 선택]
  │
  ▼
① POST /api/v1/lens/analyze (Multipart)
   Request: { file: {앨범 이미지}, srchSlct: "d" }
   Response: (음식 스캔과 동일)
  │
  └─ 식단 상세페이지 (분석 데이터 반영)
```

```
[진입 경로 3 — 음식 검색]
  │
  ▼
④ POST /api/v1/lens/search
   Request: { key: "{검색어}" }
   Response: [{ id, foodName, manufacturer }]
   → 0.3초 디바운스, 검색 결과 리스트 표시
  │
  ▼
[음식 선택]
  │
  ▼
⑤ POST /api/v1/lens/detail
   Request: { code: {선택한 음식 id}, foodName: "{표시 이름}" }
   Response: { foods: [{ foodName, srvSize, srvUnit, srvCnt,
              nutrition, foodLensId, rsltNo }],
              dietHdImgNm, dietHdImgNmUrl }
  │
  └─ 식단 상세페이지 (선택한 음식 추가)
```

### 12.2 내 음식 API

```
[내 음식 탭 진입]
  │
  ▼
⑥ POST /api/v1/myFood/list
   Request: { srchTxt: "{검색어}", lastMyFoodNo: 0 }
   Response: { foods: [{ foodName, srvSize, srvUnit, srvCnt,
              nutrition, myFoodNo }], isLast: boolean }
   → 30개 단위 커서 페이징 (lastMyFoodNo로 다음 페이지 조회)
```

```
[내 음식 선택]
  │
  ▼
⑦ POST /api/v1/myFood/sel
   Request: { myFoodNo: {선택한 내 음식 번호} }
   Response: { foods: [...], dietHdImgNm, dietHdImgNmUrl }
   → 식단 상세페이지 (선택한 음식 추가)
```

```
[내 음식 등록]
  │
  ▼
⑧ POST /api/v1/myFood/upsert
   Request: { foodName, srvSize, srvUnit, srvCnt, nutrition,
             myFoodNo: 0, srchSlct, rsltNo, dietDtNo, foodLensId }
   Response: { result: { myFoodNo } }
   → myFoodNo=0: 신규 등록 / 그 외: 수정
```

```
[내 음식 삭제]
  │
  ▼
⑨ POST /api/v1/myFood/del
   Request: { myFoodNo: {삭제할 번호} }
   Response: { code: "1" }
```

### 12.3 식단 수정 진입

```
[홈 식단 리스트 → 항목 터치]
  │
  ▼
⑩ POST /api/v1/food/diet/sel
   Request: { dietHdNo: {식단 번호} }
   Response: { dietDtCnt, dietHdNm, dietHdImgNm, dietHdImgUrl,
              dietHdNo, dietInsDate,
              foods: [{ foodName, srvSize, srvUnit, srvCnt,
                        nutrition, dietDtNo, dietHdNo, myFoodNo,
                        rsltNo, foodLensId, regSlct, position, srvCntType }] }
  │
  └─ 식단 상세페이지 (기존 데이터 표시, 수정 모드)
```

### 12.4 식단 저장 (등록/수정)

```
[완료 버튼 터치]
  │
  ▼
⑪ POST /api/v1/food/upsert/check
   Response: { code: "1" (등록 가능) / "-1" (등록 초과) / "0" (에러) }
  │
  ├─ code "1" (등록 가능)
  │   ▼
  │  ⑫ POST /api/v1/food/upsert (Multipart)
  │     Request: { dietHdNo: 0 (신규) 또는 기존값 (수정),
  │               dietHdImgNm, dietInsDate: "YYYY-MM-DD HH:mm:ss",
  │               foods: [{ foodName, srvSize, srvUnit, srvCnt,
  │                         nutrition, rsltNo, myFoodNo, dietDtNo,
  │                         regSlct, position, srvCntType, widgetYn,
  │                         foodLensId }] }
  │     Response: { code: "1" (성공) / "-1" (실패) / "0" (에러) }
  │     → 성공 시 홈 복귀 (식단 리스트 + 캘린더 dot + 대시보드 갱신)
  │
  └─ code "-1" (등록 초과)
      → 에러 메시지 표시
```

  │
  └─ 성공 시 홈 복귀 (식단 리스트 + 캘린더 dot + 대시보드 갱신)

## 13. 에러 및 빈 상태 처리

| 상황 | 처리 |
|------|------|
| 푸드렌즈 인식 실패 | 식단 이름 → "음식이 아님", 영양정보 **모든 항목 0** 상태로 진행 |
| 바코드 스캔 실패 | 카메라 뷰 재전환 + 상단 재시도 문구 표시 → **2.5초 후 자동 숨김** |
| 라벨 OCR 스캔 실패 | 카메라 뷰 재전환 + 상단 재시도 문구 표시 → **2.5초 후 자동 숨김** |
| 바코드 제품 정보 없음 | OCR 스캔 유도 팝업 |
| 음식 검색 결과 없음 | "요청하신 음식을 찾을 수가 없어요" |
| 내 음식 리스트 없음 | "등록 된 내 음식이 없어요" |
| 내 음식 검색 결과 없음 | "요청하신 음식을 찾을 수가 없어요" |
| **API 네트워크 오류** | 에러 토스트 메시지 노출 |
