# 온보딩 기획서

> Figma 소스: AI 식단 분석 서비스 MVP / 스토리보드 - 온보딩
> 작성일: 2026-04-03

---

## 1. 개요

신규 사용자의 신체 데이터를 수집하여 개인 맞춤 목표 칼로리 및 영양소를 산출하고, 회원 등록 후 메인 화면으로 진입시킨다.

**진입 조건:** 스플래시에서 미등록 사용자로 판별된 경우
**완료 조건:** 회원 등록 + 로그인 + 영양소 등록 → 홈 이동

---

## 2. 온보딩 단계 요약

| 순서  | 화면           | 프로그레스바     | 입력 항목                  |
| --- | ------------ | ---------- | ---------------------- |
| 1/5 | 성별 선택        | ████░░░░░░ | 남성 / 여성                |
| 2/5 | 출생연도 입력      | ████████░░ | 출생연도 (픽커)              |
| 3/5 | 신장·체중 입력     | ████████░░ | 키(cm), 몸무게(kg) (듀얼 픽커) |
| 4/5 | 활동량 선택       | ████████░░ | 4단계 활동량                |
| 5/5 | 목표 선택        | ██████████ | 감량 / 유지 / 증량           |
| 결과  | TDEE 결과 + 시작 | —          | 없음 (결과 확인 + 시작하기)      |

## 3. 화면별 상세 기획

### 3.1 Step 1 — 성별 선택 (1/5)

| 항목         | 내용                                  |
| ---------- | ----------------------------------- |
| **프로그레스바** | 1/5 진행도 게이지 표시                      |
| **타이틀**    | "성별을 선택해 주세요"                       |
| **서브타이틀**  | "입력한 정보는 목표 칼로리 및 영양정보 분석에 사용돼요"    |
| **입력 UI**  | 선택 버튼 2개 — "남성" / "여성"              |
| **동작**     | 버튼 터치 시 **즉시 다음 페이지 이동** (다음 버튼 없음) |
| **뒤로가기**   | 온보딩 종료 (앱 종료)                       |

### 3.2 Step 2 — 출생연도 입력 (2/5)

| 항목         | 내용                                 |
| ---------- | ---------------------------------- |
| **프로그레스바** | 2/5 진행도 게이지 표시                     |
| **타이틀**    | "출생연도를 선택해 주세요"                    |
| **서브타이틀**  | "입력한 정보는 목표 칼로리 및 영양정보 분석에 사용돼요"   |
| **입력 UI**  | 휠 픽커 (스크롤)                         |
| **설정 범위**  | 최소: 1900 / 최대: 2026                |
| **디폴트값**   | 1991 (35세)                         |
| **경계값 동작** | 최소/최대값에서 **바운스 없이 스크롤 정지**         |
| **다음 버튼**  | 항상 활성화 → 터치 시 다음 단계 이동             |
| **뒤로가기**   | 이전 단계(성별 선택)로 이동, **이전 입력 데이터 유지** |

### 3.3 Step 3 — 신장·체중 입력 (3/5)

| 항목 | 내용 |
|------|------|
| **프로그레스바** | 3/5 진행도 게이지 표시 |
| **타이틀** | "신장과 체중을 선택해주세요" |
| **서브타이틀** | "입력한 정보는 목표 칼로리 및 영양정보 분석에 사용돼요" |
| **입력 UI** | 듀얼 휠 픽커 — 좌(신장) / 우(체중), 스크롤 터치 영역 구분 |

**픽커 상세:**

| 항목 | 신장 (좌) | 체중 (우) |
|------|----------|----------|
| 최소 | 60 cm | 30 kg |
| 최대 | 220 cm | 300 kg |
| 증감 단위 | 1 cm | 1 kg |
| 디폴트 (남성) | 173 cm | 70 kg |
| 디폴트 (여성) | 160 cm | 55 kg |

| 항목 | 내용 |
|------|------|
| **햅틱** | 픽커 조절 시 햅틱 효과 적용 |
| **스크롤 제한** | 무한스크롤 아님 — 최소/최대값에서 **바운스 없이 스크롤 정지** |
| **다음 버튼** | 항상 활성화 → 터치 시 다음 단계 이동 |
| **뒤로가기** | 이전 단계(출생연도 입력)로 이동, **이전 입력 데이터 유지** |

### 3.4 Step 4 — 활동량 선택 (4/5)

| 항목 | 내용 |
|------|------|
| **프로그레스바** | 4/5 진행도 게이지 표시 |
| **타이틀** | "평소 활동량이 어떻게 되세요?" |
| **서브타이틀** | "입력한 정보는 목표 칼로리 및 영양정보 분석에 사용돼요" |
| **입력 UI** | 선택 버튼 4개 (라디오형) |

**활동량 선택지:**

| 레벨 | 라벨 | 설명 | 활동 계수 |
|------|------|------|----------|
| 1 | 극소 활동 | 하루 대부분을 앉아서 보냄, 이동 시 차량 이용 | 1.2 |
| 2 | 간헐적 활동 | 사무직이나 주 2~3회 규칙적 운동 | 1.375 |
| 3 | 생활 밀착 활동 | 판매직, 교사, 가사 노동 비중 높은 군 | 1.55 |
| 4 | 고강도 종사 | 택배기사, 건설 현장, 전문 운동선수 | 1.725 |

| 항목 | 내용 |
|------|------|
| **선택 동작** | 버튼 터치 시 선택 전환 + 하단 다음 버튼 활성화 |
| **중복 선택** | 불가 — 터치 후 재터치 시 선택 해제 |
| **다음 버튼** | 선택 전 **비활성화** → 선택 시 활성화 → 재터치 해제 시 비활성화 |
| **뒤로가기** | 이전 단계(신장·체중)로 이동, **이전 입력 데이터 유지** |

## 4. API 기반 플로우

> 영양소 계산은 서버 API에서 처리 (`POST /api/v1/nutrition/calculate`)

### 4.1 온보딩 입력 플로우 (API 호출 없음)

```
[Step 1: 성별] ──(터치 즉시)──→ [Step 2: 출생연도]
                                      │ (다음)
                                      ▼
                               [Step 3: 신장·체중]
                                      │ (다음)
                                      ▼
                               [Step 4: 활동량]
                                      │ (다음, 선택 필수)
                                      ▼
                               [Step 5: 목표]
                                      │ (다음, 선택 필수)
                                      ▼
                               [결과 화면: 로딩]
```

- Step 1~5는 클라이언트 로컬 상태만 사용 (서버 통신 없음)
- 각 단계의 입력값은 ViewModel에서 보관, 뒤로가기 시 유지

### 4.2 시작하기 API 시퀀스 (순차 호출)

```
[시작하기 버튼 터치] → 버튼 비활성화 (중복 방지)
  │
  ▼
① POST /api/v1/nutrition/calculate
   Request: sex, birthYear, heightCm, weightKg, activityLevel, goal
   Response: memBmrKcal, memTdeeKcal, memGoalEnergy,
             memGoalChocdf, memGoalProtein, memGoalFat,
             memGoalSugar, memGoalNat, memGoalFibtg,
             memGoalSfa, memGoalTrnfa, memGoalChole, memGoalWater
   실패 → 토스트 메시지 → 버튼 재활성화
  │ 성공
  ▼
② POST /api/v1/member/register
   Request: memSex ("m"/"f"), memBirthDate ("YYYY-MM-DD")
   Response: code "1" (완료) / "-11" (중복 디바이스) / "-20" (SMTP 에러)
   실패 → 토스트 메시지 → 버튼 재활성화
  │ 성공
  ▼
③ POST /api/v1/auth/login
   Request: 없음 (User-Agent 헤더의 deviceId로 식별)
   Response: Set-Cookie (JSESSIONID, remember-me)
   실패 → 토스트 메시지 → 버튼 재활성화
  │ 성공
  ▼
④ POST /api/v1/nutrition/register
   Request: memHeight, memWeight, memExSlct, memGoalSlct,
            memBmrKcal, memTdeeKcal, memGoalEnergy,
            memGoalChocdf, memGoalProtein, memGoalFat,
            memGoalSugar, memGoalNat, memGoalFibtg,
            memGoalSfa, memGoalTrnfa, memGoalChole, memGoalWater
   실패 → 토스트 메시지 → 버튼 재활성화
  │ 성공
  ▼
[홈 화면 이동]
```

### 4.3 API 엔드포인트 요약

| 순서 | API | Method | 핵심 입력 | 핵심 출력 |
|------|-----|--------|----------|----------|
| ① | /api/v1/nutrition/calculate | POST | 성별, 나이, 키, 몸무게, 활동량, 목표 | BMR, TDEE, 목표 칼로리, 탄/단/지, 부가 영양소 |
| ② | /api/v1/member/register | POST | memSex, memBirthDate | 등록 결과 코드 |
| ③ | /api/v1/auth/login | POST | — (deviceId 기반) | 세션 쿠키 |
| ④ | /api/v1/nutrition/register | POST | 신체정보 + ①의 계산 결과 전체 | 응답 코드 |

## 5. 인터랙션 정의

### 5.1 프로그레스바

| 항목 | 내용 |
|------|------|
| 위치 | 화면 상단 |
| 표시 | N/5 진행도 게이지 (채움 애니메이션) |
| 전환 | 단계 이동 시 300ms ease-out 애니메이션 |

### 5.2 페이지 전환

| 항목 | 내용 |
|------|------|
| 전진 | 우 → 좌 슬라이드 (300ms) + 다음 화면 50ms 딜레이 |
| 후퇴 (뒤로가기) | 좌 → 우 슬라이드 (300ms) |

### 5.3 버튼 인터랙션

| 버튼 유형 | 동작 |
|----------|------|
| 성별 선택 | 터치 즉시 다음 페이지 이동 |
| 활동량/목표 선택 | 터치 시 선택 전환 + 다음 버튼 활성화 |
| 다음 버튼 | 항상 활성화 상태 유지 |
| 시작하기 | 스크롤 하단 위치, 터치 시 API 호출 시퀀스 시작 |

### 5.4 픽커 인터랙션

| 항목 | 내용 |
|------|------|
| 출생연도 픽커 | 단일 휠, 무한스크롤 아님 |
| 듀얼 픽커 (신장·체중) | 좌/우 독립 스크롤, 터치 영역 구분 |
| 햅틱 | 픽커 값 변경 시 햅틱 효과 적용 |
| 스크롤 제한 | 최소/최대값에서 스크롤 정지 |

### 5.5 결과 화면 애니메이션

| 항목 | 내용 |
|------|------|
| 반도넛 차트 | 좌측 → 우측 방향 채움 애니메이션 (500ms) |
| 로딩 → 결과 전환 | 계산 완료 후 결과 화면 페이드인 |

                                          ▼
                                   [Step 3: 신장·체중]
                                          │

                                   [Step 4: 활동량]
                                          │

                                   [Step 5: 목표]

                                          ▼
                                   [결과: TDEE 산출]
                                          │
                                    (시작하기)
                                          ▼

| Step 4 (활동량) | Step 3 (신장·체중) | 신장/체중 값 유지 |
