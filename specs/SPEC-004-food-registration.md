## 1. 목적

홈 화면 FAB를 통해 새로운 식단을 등록한다. 사진 찍기(음식 스캔/바코드/라벨 OCR) / 앨범 선택 / 음식 검색의 3가지 진입 경로를 제공하며, 모든 경로는 식단 상세페이지(SPEC-005)로 수렴한다.

---

## 2. 용어 정의

| 용어 | 정의 |
|------|------|
| 식단 | 집합. 아침·점심 등 끼니를 나타내는 상위 컨테이너. 하나 이상의 음식으로 구성 |
| 음식 | 원소. 식단에 포함된 개별 메뉴. 고유의 영양 데이터를 가짐 |
| 푸드렌즈 | 음식 이미지를 분석하여 음식명과 영양정보를 반환하는 AI 분석 API |
| OCR | Optical Character Recognition. 영양성분 라벨 이미지를 텍스트로 추출하는 기능 |
| 내 음식 | 사용자가 즐겨찾기로 등록해 둔 개인 음식 목록 |
| srchSlct | 검색 경로 구분자 ("a": 촬영, "d": 앨범) |
| rsltNo | 푸드렌즈 분석 결과 고유 번호 |
| myFoodNo | 내 음식 고유 번호 |

---

## 3. 전제

- 홈 화면(SPEC-003)의 FAB 터치 또는 빈 식단 리스트 터치로 진입한다.
- 모든 진입 경로의 최종 도착지는 식단 상세페이지(SPEC-005)다.
- 등록 시간은 캘린더 설정 일자 기준 오후 11:59이며, 앨범 선택(당일)에만 예외가 존재한다.

---

## 4. 외부 의존성

| 이름 | 유형 | 용도 |
|------|------|------|
| POST /api/v1/lens/analyze (Multipart) | REST API | 음식 이미지 푸드렌즈 분석 |
| POST /api/v1/foodsafety/barcode | REST API | 바코드 기반 제품 정보 조회 |
| POST /api/v1/ocr (Multipart) | REST API | 영양성분 라벨 OCR 추출 |
| POST /api/v1/lens/search | REST API | 음식명 텍스트 검색 |
| POST /api/v1/lens/detail | REST API | 선택한 음식의 상세 영양정보 조회 |
| POST /api/v1/myFood/sel | REST API | 내 음식 개별 상세 조회 |
| 카메라 SDK | SDK | 사진 촬영 및 바코드/OCR 스캔 |
| 시스템 앨범 피커 | SDK | 앨범 사진 선택 |

## 5. 행위 명세

### 5.1 진입 옵션 (FAB 바텀시트)

| 옵션 | 동작 |
|------|------|
| 사진 찍기 | 카메라 권한 체크 → 카메라 뷰 (3모드 탭) |
| 앨범 선택 | 앨범 권한 체크 → 시스템 앨범 피커 |
| 음식 검색 | 음식 검색 페이지 이동 |

### 5.2 진입 경로 1 — 사진 찍기

**카메라 권한 체크:**

| 상태 | 동작 |
|------|------|
| 미정 (최초) | 시스템 권한 팝업 노출 |
| 허용 | 즉시 카메라 실행 |
| 거절 | 설정 유도 팝업 → "식단 사진을 촬영하려면 카메라 권한이 필요해요. 설정에서 권한을 허용해 주세요." → 확인 시 앱 설정 카메라 권한 화면 이동 |

**카메라 모드 탭:** 바코드 / 음식 스캔(디폴트) / 라벨

**음식 스캔 (푸드렌즈):**
1. 촬영 → POST /api/v1/lens/analyze (Multipart) { file, srchSlct: "a" }
2. 로딩 로티 표시 (필요 시)
3. 인식 성공 → 식단 상세페이지 이동 (음식명 + 영양정보 자동 입력)
4. 인식 실패 → 식단 상세페이지 이동 (음식명: "음식이 아님", 영양정보: 0)

**바코드 스캔:**
1. 바코드 인식 → 즉시 POST /api/v1/foodsafety/barcode { barcode }
2. 제품 정보 있음 → 식단 상세페이지 이동 (바코드 데이터 반영)
3. 제품 정보 없음 → OCR 유도 팝업: "대신 영양성분표를 사진으로 찍으면 영양 정보를 자동으로 읽어올 수 있어요!"
4. 유도 팝업 → 스캔하기 터치 → 라벨 모드 전환
5. 스캔 실패 → 카메라 뷰 재전환 + 상단 재시도 문구 (2.5초 후 비노출)

**영양정보 라벨 OCR:**
1. 촬영 → POST /api/v1/ocr (Multipart) { file }
2. 로딩 로티 표시 (필요 시)
3. 성공 → OCR 결과 확인 페이지 이동
4. 실패 → 카메라 뷰 재전환 + 상단 재시도 문구 (2.5초 후 비노출)

**OCR 결과 확인 페이지:**
- 음식명 입력 (필수, 1~20자, 특수문자 가능, 이모지 불가)
- 공백 처리: 앞뒤 공백 제거, 연속 공백 단일 공백 치환, 공백만 입력 시 내용 없음 처리
- 영양정보 인풋 (칼로리/탄수화물/단백질/지방/당류/나트륨/식이섬유/콜레스테롤/포화지방)
- 저장 버튼: 음식명 입력 시 활성화 → 터치 시 식단 상세페이지 이동

**영양정보 입력 정책:**

| 항목 | 정책 |
|------|------|
| 키보드 | 숫자 입력 키보드 |
| 소수점 | 첫째 자리까지 (0 가능) |
| 칼로리/내용량 | 정수만 |
| 최대 자릿수 | 4자리 (예: 9999, 999.9) |
| 빈값 포커스 해제 | 기존 값으로 리셋 |

### 5.3 진입 경로 2 — 앨범 선택

**앨범 권한 체크:**

| 상태 | 동작 |
|------|------|
| 미정 (최초) | 시스템 권한 팝업 노출 |
| 허용 | 시스템 앨범 피커 실행 |
| 거절 | 설정 유도 팝업 → "식단 사진을 불러오기 위해 앨범 접근 권한이 필요해요." → 확인 시 앱 설정 앨범 권한 화면 이동 |

**앨범 플로우:**
1. 시스템 앨범 피커 실행 (중복 선택 불가)
2. 사진 선택 → 앨범 레이어 내림
3. 등록 시점이 당일인 경우에만 시간 기준 선택 팝업 출력
4. 시간 선택 → POST /api/v1/lens/analyze { file, srchSlct: "d" } → 식단 상세페이지

**시간 기준 선택 팝업:**
- 문구: "사진이 촬영된 날짜로 기록할지, 현재 시간 기준으로 기록할지 선택해 주세요."
- 촬영시간: 사진 메타데이터 기준 일자/시간
- 현재시간: 현재 디바이스 시간
- 당일 아닌 경우: 팝업 없음, 오후 11:59 기준 자동 처리

### 5.4 진입 경로 3 — 음식 검색

**검색 페이지:**
- 진입 시 인풋창 활성화 (키보드 자동 실행)
- 등록 시간: 캘린더 설정 일자 기준 오후 11:59

**검색 인풋:**
- 최대 20자
- 1글자 이상 입력 시 텍스트 전체 삭제(x) 버튼 노출
- 스크롤 시 키보드 내림

**최근 검색 음식:**
- 인풋 비어있을 때 표시, 입력 시 비노출
- 최대 10개 (11번째 검색 시 가장 오래된 항목 자동 제거)
- 음식명 칩 디자인, 최근 선택순 정렬
- 터치 시 해당 검색어 자동 입력 + API 즉시 호출

**검색 결과 (음식 검색 탭):**
- POST /api/v1/lens/search { key } 호출 (0.3초 디바운스, 앞뒤 공백 제거, 최소 1글자)
- 표시: 음식명 + 제조사 (없으면 미표시)
- 결과 없음: "요청하신 음식을 찾을 수가 없어요"
- 음식 선택 → POST /api/v1/lens/detail { code, foodName } → 식단 상세페이지

**내 음식 탭:**
- 리스트 정렬: 최근 등록순, 30개 단위 페이징
- 검색: 0.3초 디바운스
- 빈 리스트: "등록 된 내 음식이 없어요"
- 검색 결과 없음: "요청하신 음식을 찾을 수가 없어요"
- 음식 선택 → POST /api/v1/myFood/sel { myFoodNo } → 식단 상세페이지
- 내 음식 추가 버튼 → 내 음식 등록 페이지

**탭 전환 (음식 검색 ↔ 내 음식):**
- 각 탭 인풋 값 별도 관리 (탭 전환 시 기존 입력값 유지)

### 5.5 내 음식 등록 페이지

- 음식명: 필수, 1~20자, 특수문자 가능, 이모지 불가, 앞뒤 공백 제거, 연속 공백 단일 치환
- 영양정보: 위 5.2 OCR 결과 확인과 동일한 입력 정책 적용
- 영양성분 라벨 인식 버튼: 카메라 실행 (권한 체크) → OCR 추출 → 인풋 자동 반영
- 저장 버튼: 음식명 입력 시 활성화 → 저장 후 내 음식 탭으로 복귀

### 5.6 등록 시간 정책

| 조건 | 등록 시간 |
|------|----------|
| 캘린더 설정 일자 기준 (기본) | 해당 일자 오후 11:59 |
| 앨범 선택 (당일) | 촬영시간 또는 현재시간 중 사용자 선택 |
| 앨범 선택 (당일 외) | 오후 11:59 자동 처리 |

### 5.7 필수값 / 선택값

| 구분 | 항목 | 정책 |
|------|------|------|
| 필수 | 식단 등록 시간·날짜 | YYYY-MM-DD HH:mm |
| 필수 | 음식명 | 빈 문자열 불가 |
| 필수 | 기준 내용량 | 음수 불가, 0 가능 |
| 필수 | 칼로리 및 영양소 | 음수 불가, 0 가능 |
| 선택 | 음식 사진 | 없으면 대체 일러스트 출력 |
| 선택 | 부가 영양소 | 없으면 0으로 표시 |

## 6. 에러 처리

| 에러 상황 | 복구 전략 | 사용자 피드백 | 로깅 |
|----------|----------|-------------|------|
| 카메라 권한 거절 | 화면 이동: 앱 설정 | 다이얼로그 | warn |
| 앨범 권한 거절 | 화면 이동: 앱 설정 | 다이얼로그 | warn |
| 푸드렌즈 분석 실패 | 무시 (0값으로 진행) | 없음 | warn |
| 바코드 스캔 실패 | 카메라 뷰 재전환 | 인라인 에러 (2.5초) | warn |
| OCR 스캔 실패 | 카메라 뷰 재전환 | 인라인 에러 (2.5초) | warn |
| 음식 검색 API 실패 | 무시 | 토스트 | error |
| 세션 만료 (401) | 화면 이동: 로그인 | 없음 | warn |
| 네트워크 오류 | 무시 | 토스트 | warn |

## 7. 범위

**포함:**
- FAB 바텀시트 (3가지 진입 옵션)
- 사진 찍기: 음식 스캔 / 바코드 / 라벨 OCR
- 앨범 선택: 권한 체크, 시간 기준 선택 팝업
- 음식 검색: 텍스트 검색, 최근 검색, 내 음식 탭
- 내 음식 등록 페이지
- OCR 결과 확인 페이지
- 권한 체크 (카메라/앨범)

**제외:**
- 식단 상세페이지 (SPEC-005)
- 식단 수정 (SPEC-005에서 처리)
- 홈 화면 (SPEC-003)
