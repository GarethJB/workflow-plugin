## 1. 목적

식단 등록/수정의 핵심 화면. 모든 진입 경로(촬영/앨범/검색/기존 식단 터치)가 수렴하며, 음식의 추가·삭제·수량 변경·영양정보 수정과 식단 저장/삭제를 수행한다.

---

## 2. 용어 정의

| 용어 | 정의 |
|------|------|
| 식단 | 집합. 하나 이상의 음식으로 구성되는 상위 컨테이너 |
| 음식 | 원소. 식단에 포함된 개별 메뉴. 고유의 영양 데이터를 가짐 |
| 식단 수량 | 한 끼니 전체의 섭취 비율 (예: 0.5인분) |
| 음식 수량 | 개별 음식의 섭취량 (수량형: 소수, 비율형: 분수) |
| 1기준 영양소 | 수량 1.0 기준의 해당 음식 영양소 값 |
| 내 음식 | 사용자가 즐겨찾기로 등록해 둔 개인 음식 목록 |
| dietHdNo | 식단 헤더 번호 (식단 고유 식별자) |
| dietDtNo | 식단 상세 번호 (음식 고유 식별자) |
| srvCntType | 수량 입력 타입 (수량형 / 비율형) |

---

## 3. 전제

- 식단의 총 칼로리 및 영양정보는 음식들의 합산 파생값이며, 직접 수정 불가.
- 음식 추가/삭제/변경은 즉시 식단 영양정보에 반영된다.
- 내 음식 등록/해제 상태는 동일 음식이 존재하는 모든 식단에 걸쳐 공유된다.

## 5. 행위 명세

### 5.1 상단 영역

| 항목 | 정책 |
|------|------|
| 타이틀 (단일 음식) | 해당 음식명 |
| 타이틀 (복수 음식) | `{가장 고칼로리 음식명} 외 {N}개` |
| 타이틀 갱신 조건 | 음식 추가/삭제/칼로리 수정 시 고칼로리 기준 재산정 |
| 더보기 버튼 | 터치 시 버튼 리스트 출력 |
| 삭제 메뉴 | 기록 완료(저장) 전 비노출, 완료 후에만 노출 |
| 식단 사진 | 업로드 사진 + 분석 라벨 표시 / 사진 없으면 대체 이미지 |
| 등록 시간 | 식단 등록 시간 표시 |

### 5.2 영양정보 요약 (수정 불가)

| 항목 | 정책 |
|------|------|
| 표시 내용 | 총 칼로리 + 탄/단/지 |
| 탄/단/지 게이지 | 비율에 맞춰 게이지 길이 연동 표시 |
| 수정 가능 여부 | 수정 불가 (음식 합산 파생값) |
| 최대 표시 수치 | 99,999 (초과 시 99,999 고정) |
| 스크롤 | 스크롤 시 2페이지로 부가 영양소(당류/나트륨/식이섬유/콜레스테롤) 표시 |

### 5.3 식단 수량 수정

- 용도: 한 끼니 전체 섭취 비율 조정
- 인풋 터치 → 숫자 키보드 출력
- 소수점 둘째 자리까지, 0 불가, 최대 4자리 (예: 99.99)
- 빈값 포커스 해제 시 기존 값으로 리셋
- -/+ 버튼: 0.1~1 구간은 0.1씩 / 1 초과는 1씩 증감

**계산 로직:**
```
Step 1: 식단 내 모든 음식의 수량 1 기준 영양소 합산
Step 2: (1기준 영양소) × 입력 수량 → 각 영양소 산출
Step 3: 칼로리 → 소수점 첫째 자리 반올림(정수) / 탄단지 → 소수점 둘째 자리 반올림(소수 한 자리)
```

### 5.4 음식 리스트

| 항목 | 정책 |
|------|------|
| 표시 | 음식명 / 중량 / 칼로리 |
| 정렬 | 고칼로리순 |
| 내 음식 뱃지 | 내 음식으로 등록된 항목에 뱃지/아이콘 표시 |
| 터치 | 음식 수정 페이지 이동 |
| 최대 개수 | 20개 (도달 시 추가 버튼 비노출) |

### 5.5 음식 삭제

| 단계 | 동작 |
|------|------|
| 1 | 삭제 버튼(X) 터치 → "음식 삭제" 버튼으로 전환 |
| 2 | 전환 후 이외 영역 터치 또는 3초간 터치 미발생 → 기존 버튼으로 원복 |
| 3 | "음식 삭제" 터치 → 해당 음식 삭제 + 식단명/라벨/칼로리·탄단지 갱신 |

- 음식 리스트 1개일 경우 삭제 버튼 비노출

### 5.6 음식 추가

- 추가 버튼 터치 → 음식 검색 페이지 이동 (SPEC-004 음식 검색과 동일 로직)
- 추가 시 식단에 음식 추가 + 식단 영양정보 갱신
- 중복 음식 추가 (푸드렌즈 검색 API 통해 추가한 동일 음식 재추가): 기존 동일 음식 수량 +1 + 영양정보 수정사항 초기화

### 5.7 음식 수정 페이지

**화면 구성:**
- 음식 이름 / 칼로리·총 내용량 / 탄단지 비율 차트 / 영양정보 9항목

**음식 수량 수정 — 수량 타입 (디폴트):**
- 소수점 둘째 자리까지, 0 불가, 최대 4자리
- 빈값 포커스 해제 시 기존값 리셋
- -/+ 버튼: 0~1 구간 0.1씩, 1 초과 1씩 증감
- 0 이하 소수점에서 1 이상 구간으로 증가 시 1로 전환

**음식 수량 수정 — 비율 타입 (듀얼 픽커):**
- 정수 픽커: 0~99 (1씩)
- 분수 픽커: 0, 1/8, 1/4, 3/8, 1/2, 5/8, 3/4, 7/8
- -/+ 버튼: 1/8씩 증감

**타입 전환 변환 테이블:**

비율 → 수량:

| 분수 | 소수 |
|------|------|
| 1/8 | 0.13 |
| 1/4 | 0.25 |
| 3/8 | 0.38 |
| 1/2 | 0.50 |
| 5/8 | 0.63 |
| 3/4 | 0.75 |
| 7/8 | 0.88 |

수량 → 비율:

| 소수 범위 | 분수 |
|----------|------|
| .00~.06 | 0 |
| .07~.18 | 1/8 |
| .19~.31 | 1/4 |
| .32~.43 | 3/8 |
| .44~.56 | 1/2 |
| .57~.68 | 5/8 |
| .69~.81 | 3/4 |
| .82~.93 | 7/8 |
| .94~.99 | 정수+1, 분수 0 |

**수량 변경 계산 로직:**
```
(1기준 영양소) × 입력 수량 → 새 영양소
반올림: 칼로리 → 정수 / 탄단지 → 소수 한 자리
단일 반올림 원칙: 최종 산출물에서만 한 번 반올림
```
완료 버튼 터치 또는 키보드 종료 시 영양정보 반영.

**칼로리 직접 수정:**
- 정수만, 0 가능, 최대 4자리 (9999), 포커스 해제 시 반영
```
배수 = 수정 칼로리 / 원본 1.0 기준 칼로리 (소수점 4자리 내부 사용)
새 영양소 = 원본 1.0 기준 영양소 × 배수 → 소수점 둘째 자리 반올림(소수 한 자리)
예외: 기존 탄/단/지 합계가 0 또는 null → 칼로리만 단독 변경
```

**탄/단/지 직접 수정:**
- 소수점 첫째 자리까지, 0 가능, 최대 4자리 (999.9), 포커스 해제 시 반영
```
변동분(Diff) = 입력값 - 수정 전 값
칼로리 환산: 탄수화물/단백질 × 4 kcal, 지방 × 9 kcal
최종 칼로리 = 기존 칼로리 + 환산값 → 소수점 첫째 자리 반올림(정수)
단일 반올림 원칙 적용
제한: 계산 결과 9999 초과 시 9999 고정
```

**부가 영양소 수정 (당류/나트륨/식이섬유/트랜스지방/포화지방/콜레스테롤):**
- 소수점 첫째 자리까지, 0 가능, 최대 4자리, 포커스 해제 시 반영
- 칼로리 연동 없음 (독립적)

**초기화 / 수정완료 버튼:**
- 초기화: 모든 인풋 직전 값으로 초기화
- 수정완료: 식단 상세페이지로 전환 + 데이터 변동 반영

### 5.8 수정 계산 로직 요약

| 케이스 | 입력 | 반올림 |
|--------|------|--------|
| 식단 수량 | 소수 2자리, 0 불가, 4자리 | 칼로리: 정수 / 탄단지: 소수1 |
| 음식 수량 | 소수 2자리 or 분수, 0 불가 | 칼로리: 정수 / 탄단지: 소수1 |
| 칼로리 수정 | 정수, 0 가능, 4자리 | 탄단지: 소수1 (배수 내부 소수4) |
| 탄단지 수정 | 소수 1자리, 0 가능, 4자리 | 칼로리: 정수 |

**단일 반올림 원칙:** 최종 산출물에서만 단 한 번 반올림 수행

### 5.9 내 음식 등록 토글

| 상태 전환 | 동작 | 토스트 메시지 |
|----------|------|-------------|
| 비활성 → 활성 | POST /api/v1/myFood/upsert → 내 음식 리스트 등록 | "내 음식에 저장되었습니다!" (2.5초) |
| 활성 → 비활성 | POST /api/v1/myFood/del → 내 음식 리스트 제거 | "내 음식에서 삭제되었습니다." (2.5초) |
| 토스트 출력 중 | 버튼 작동 방지 | — |

- 지원 소스: 직접 등록 / OCR / 푸드렌즈 / 바코드 모두 지원
- 크로스 식단 상태 공유: 동일 음식이 다른 식단에 존재하면 내 음식 상태 일관 공유

### 5.10 식단 삭제 (더보기 메뉴)

- 노출 조건: 기록 완료(저장) 후에만 더보기 메뉴에 노출
- "식단 기록을 삭제할까요?" / "삭제한 기록은 다시 되돌릴 수 없어요"
- 확인 시 → POST /api/v1/food/head/del → 홈 복귀

### 5.11 완료 (저장)

1. 완료 버튼 터치
2. POST /api/v1/food/upsert/check → code "1" (가능) / "-1" (초과) / "0" (에러)
3. code "1" → POST /api/v1/food/upsert (Multipart) → 성공 시 홈 복귀 (식단 리스트 + 캘린더 dot + 대시보드 갱신)
4. code "-1" → 에러 메시지 표시

### 5.12 뒤로가기 (최초 저장 전)

- 미저장 상태에서 뒤로가기 → 작성 취소 컨펌 출력
- "작성 취소" → 홈 복귀 + 식단 페이지 소멸
- "아니오" → 컨펌 닫기, 식단 상세 유지

## 6. 에러 처리

| 에러 상황 | 복구 전략 | 사용자 피드백 | 로깅 |
|----------|----------|-------------|------|
| 식단 조회 API 실패 | 무시 | 토스트 | error |
| 저장 가능 여부 확인 실패 (code "0") | 버튼 재활성화 | 토스트 | error |
| 저장 초과 (code "-1") | 버튼 재활성화 | 인라인 에러 | warn |
| 식단 저장 API 실패 (code "-1") | 버튼 재활성화 | 토스트 | error |
| 식단 삭제 API 실패 | 무시 | 토스트 | error |
| 내 음식 등록 API 실패 | 버튼 재활성화 | 토스트 | error |
| 내 음식 삭제 API 실패 | 버튼 재활성화 | 토스트 | error |
| 세션 만료 (API 401) | 화면 이동: 로그인 | 없음 | warn |
| 네트워크 오류 | 무시 | 토스트 | warn |
