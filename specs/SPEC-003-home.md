## 1. 목적

온보딩 또는 로그인 완료 후 진입하는 앱의 메인 화면. 선택한 날짜의 식단·영양소·수분 섭취 현황을 표시하고, 식단 등록/삭제/상세 조회를 수행한다.

---

## 2. 용어 정의

| 용어 | 정의 |
|------|------|
| 대시보드 | 칼로리·영양소·수분 섭취 현황을 시각화하는 3페이지 스와이프 영역 |
| 권장 칼로리 | 온보딩 또는 정보 수정 시 산출된 일일 목표 칼로리 |
| dot indicator | 캘린더에서 해당 날짜에 식단 기록이 있음을 나타내는 시각적 표시 |
| FAB | Floating Action Button. 우측 하단에 고정된 식단 등록 진입 버튼 |
| TDEE | Total Daily Energy Expenditure. 활동량 반영 일일 총 에너지 소비량 |
| dietHdNo | 식단 헤더 번호 (식단 고유 식별자) |

---

## 3. 전제

- 사용자가 온보딩 또는 로그인을 완료한 상태에서 진입한다.
- 권장 칼로리는 사용자 정보 수정 이후 날짜부터 반영되며, 과거 날짜에 소급 적용되지 않는다.
- 캘린더 표시 범위 시작일은 2025년 1월 1일이다.

## 4. 외부 의존성

| 이름 | 유형 | 용도 |
|------|------|------|
| POST /api/v1/daily/getCalendar | REST API | 캘린더 dot indicator 및 권장 초과 여부 조회 |

| POST /api/v1/water/ins | REST API | 수분 섭취량 저장 |
| POST /api/v1/food/head/del | REST API | 식단 삭제 |

## 5. 행위 명세

### 5.1 전역 정책

| 항목 | 정책 |
|------|------|
| 숫자 표기 | 페이지 내 모든 숫자는 정수 표현 (소수점 첫째 자리에서 반올림) |
| 천 단위 | 만/천 단위 콤마 표시 |
| FAB | 우측 하단 항상 고정 표시 |

### 5.2 데이터 갱신 시점

| 트리거 | 호출 API |
|--------|----------|
| 화면 최초 진입 | ①②③④ 전체 |
| 날짜 선택 | ②③④ |
| 식단 등록/삭제 후 | ①②③ |
| 디바이스 날짜 변동 (자정 경과) | ①②③④ 전체 |
| 탭 이동 후 복귀 | ②③④ |
| 앱 백그라운드→포그라운드 전환 | ②③④ |

### 5.3 캘린더 영역

- 날짜 표기: 당해 연도 `M. D 요일` / 지나간 연도 `YY. M. D 요일`
- 좌/우 스크롤로 날짜 이동 (요일은 고정), 월 초과 시 바운스 효과
- 미래 날짜(오늘 이후) 터치(이동) 불가
- 오늘 외 날짜 조회 중: 오늘 이동 아이콘 노출 → 터치 시 당일 이동 및 데이터 새로고침
- 식단 기록 있는 날짜: dot indicator 표시
- 권장 칼로리 초과 날짜: 별도 컬러 강조
- 요일 표기: 오늘 → "오늘" / 그 외 → 요일명

### 5.4 대시보드 — 1페이지 (칼로리 및 탄/단/지)

**반도넛 차트:**
- 권장 대비 섭취 칼로리 진행률 시각화 (좌→우 방향)
- 100% 초과 시 게이지 모두 채워진 형태 유지

**섭취 칼로리 표시:**
- 최대 5자리 (0~99,999), 초과 시 99,999 고정

**칼로리 가이드 문구:**

| 조건 | 문구 |
|------|------|
| 식단 미등록 | "오늘은 어떤 음식을 드셨나요?" |
| 권장 칼로리 미만 | "권장 칼로리까지 {남은칼로리}kcal 남았어요" |
| 권장 칼로리 정확히 충족 | "권장 칼로리를 정확히 채웠어요!" |
| 권장 칼로리 초과 | "권장 칼로리를 {초과칼로리}kcal 초과했어요" |

**탄/단/지 표시:**
- 섭취량 최대 5자리 (0~99,999), 초과 시 99,999 고정
- 권장량 최대 4자리 (0~9,999), 초과 시 9,999 고정
- 전체 섭취 대비 비율(%) 표시

**상태 라벨 기준:**

| 영양소 | 부족 | 적정 | 과다 |
|--------|------|------|------|
| 탄수화물 | < 80% | 80~120% | > 120% |
| 단백질 | < 90% | 90~130% | > 130% |
| 지방 | < 80% | 80~110% | > 110% |

### 5.5 대시보드 — 2페이지 (부가 영양소)

**제한 권장 영양소 (당류/나트륨/콜레스테롤):**
- 막대 그래프: 권장량 대비 0~1.5배 범위 표현, 1.5배 초과 시 최대치 고정
- 상태 라벨: 적정(100% 이하) / 주의(100~120%) / 위험(120% 초과)

**섭취 권장 영양소 (식이섬유):**
- 막대 그래프: 권장량 대비 0~100% 범위 표현
- 상태 라벨: 부족(100% 미만) / 적정(100% 이상)

- 날짜 변경 시 사용자가 보고 있던 대시보드 페이지 유지

### 5.6 대시보드 — 3페이지 (음수량)

- 표시: 섭취량 / 권장량 (소수점 둘째 자리까지, 예: `0L / 2.5L`)
- 상한: 5L (도달 시 + 버튼 미출력) / 하한: 0L (도달 시 - 버튼 미출력)
- (+)/(-) 버튼: 0.25L(250ml) 단위 증감, 즉시 UI 반영
- 버튼 터치 시 햅틱 효과 (0L/5L 경계 도달 시 버튼 미출력으로 햅틱 미발생)
- 빠른 연속 터치 시 각 터치마다 0.25L씩 즉시 반영
- API 호출: 500ms 디바운스 후 마지막 값 기준 1회 저장

**보조 문구:**

| 조건 | 문구 |
|------|------|
| 음수 기록 없음 | "지금 물 한 모금 어때요?" |
| 음수 기록 있음 | "조금만 더 마셔도 좋아요" |
| 권장치 도달 | "오늘 필요한 물을 충분히 마셨어요" |

**컵 수위:**
- 권장량 미달: 섭취량에 비례하여 수위 표시
- 권장량 도달/초과: 컵 수위 최대치 유지 (수치는 실제 섭취량 누적 표시)

### 5.7 식단 리스트

- 정렬: 최근 등록순 (동일 시간 등록 시 등록 선순)
- 등록 개수 제한 없음
- 리스트 비어있을 경우: 빈 상태 영역 + "오늘 먹은 음식을 기록해 보세요" 표시, 터치 시 FAB와 동일한 바텀시트 출력

**리스트 아이템 구성:**

| 구성요소 | 정책 |
|---------|------|
| 썸네일 | 식단 사진 / 없으면 대체 이미지 |
| 등록 시간 | "오전/오후" + 12시간제 (예: `오전 11:32`) |
| 식단명 (단일 음식) | 음식명 |
| 식단명 (복수 음식) | `{메인음식명} 외 {N}개` |
| 영양정보 | 칼로리(kcal) + 탄수화물(g) + 단백질(g) + 지방(g) |

- 리스트 각 항목의 합계 = 대시보드 표시값 일치

### 5.8 식단 삭제 (리스트에서)

1. 식단 블록 왼쪽으로 밀기 → 우측에 삭제 박스 출력
2. 삭제 박스 외 부분 터치(or 스크롤) → 닫힘
3. 삭제 박스 터치 → 컨펌 다이얼로그 출력
4. "삭제할래요" → POST /api/v1/food/head/del → 식단 리스트 + 대시보드 + 캘린더 dot 갱신

**컨펌 다이얼로그:**
- 타이틀: "식단 기록을 삭제할까요?"
- 서브타이틀: "삭제한 기록은 다시 되돌릴 수 없어요"
- 좌측: "아니요" → 닫기 / 우측: "삭제할래요" → 삭제 처리

### 5.9 FAB (식단 등록 진입)

- 터치 시 입력 옵션 바텀시트 노출
- 옵션: 사진 찍기 / 앨범 선택 / 음식 검색
- 입력 옵션 외 영역 터치 또는 FAB 재터치 → 바텀시트 닫기

## 10. 기술 설계

- 아키텍처: MVI + Clean Architecture

- 로컬 저장소: DataStore
