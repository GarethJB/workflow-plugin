## 1. 목적

신규 사용자의 신체 데이터(성별/출생연도/신장/체중/활동량/목표)를 수집하여 개인 맞춤 목표 칼로리 및 영양소를 산출하고, 회원 등록 후 메인 화면으로 진입시킨다.

---

## 2. 용어 정의

| 용어 | 정의 |
|------|------|
| TDEE | Total Daily Energy Expenditure. 활동량을 반영한 일일 총 에너지 소비량 |
| BMR | Basal Metabolic Rate. 기초대사량 |
| 활동 계수 | 활동량 레벨에 따라 BMR에 곱하는 배수 (1.2 ~ 1.725) |
| 목표 칼로리 | TDEE에 목표(감량/유지/증량)에 따른 계수를 곱한 일일 권장 섭취 칼로리 |
| 듀얼 픽커 | 신장과 체중을 좌/우 독립 스크롤로 입력하는 UI 컴포넌트 |
| memSex | 회원 성별 식별자 ("m" / "f") |

---

## 3. 전제

- 스플래시/로그인(SPEC-001)에서 미등록 사용자로 판별된 이후 진입한다.
- socialSlct, socialKey가 온보딩으로 전달된다.
- 영양소 계산은 클라이언트가 아닌 서버 API(POST /api/v1/nutrition/calculate)에서 처리한다.
- 온보딩 진행 중 앱 종료 후 재진입 시 Step 1(성별 선택)부터 재시작한다 (상태 미저장).

---

## 4. 외부 의존성

| 이름 | 유형 | 용도 |
|------|------|------|
| POST /api/v1/nutrition/calculate | REST API | 신체 정보 기반 BMR/TDEE/목표 영양소 산출 |
| POST /api/v1/member/register | REST API | 회원 등록 |
| POST /api/v1/auth/login | REST API | 등록 후 자동 로그인 및 세션 쿠키 발급 |
| POST /api/v1/nutrition/register | REST API | 계산된 목표 영양소 및 신체 정보 저장 |
| DataStore | 로컬 저장소 | 세션 쿠키 저장 |

## 5. 행위 명세

### 5.1 단계 요약

| 순서 | 화면 | 입력 항목 | 다음 버튼 |
|------|------|----------|----------|
| 1/5 | 성별 선택 | 남성 / 여성 | 없음 (선택 즉시 이동) |
| 2/5 | 출생연도 입력 | 휠 픽커 (1900 ~ 2026, 디폴트: 1991) | 항상 활성화 |
| 3/5 | 신장·체중 입력 | 듀얼 휠 픽커 | 항상 활성화 |
| 4/5 | 활동량 선택 | 4단계 라디오 | 선택 시 활성화 |
| 5/5 | 목표 선택 | 감량/유지/증량 라디오 | 선택 시 활성화 |
| 결과 | TDEE 결과 + 시작하기 | — | 스크롤 하단 |

### 5.2 Step 1 — 성별 선택

- "남성" / "여성" 버튼 터치 시 즉시 Step 2로 이동 (다음 버튼 없음)
- 뒤로가기: 온보딩 종료 (앱 종료)

### 5.3 Step 2 — 출생연도 입력

- 단일 휠 픽커, 범위: 1900 ~ 2026, 무한스크롤 아님
- 경계값에서 바운스 없이 스크롤 정지
- 뒤로가기: Step 1로 이동, 이전 입력값 유지

### 5.4 Step 3 — 신장·체중 입력

| 항목 | 신장 (좌) | 체중 (우) |
|------|----------|----------|
| 범위 | 60 ~ 220 cm | 30 ~ 300 kg |
| 증감 단위 | 1 cm | 1 kg |
| 디폴트 (남성) | 173 cm | 70 kg |
| 디폴트 (여성) | 160 cm | 55 kg |

- 좌/우 독립 스크롤, 터치 영역 구분
- 픽커 값 변경 시 햅틱 효과 적용
- 경계값에서 바운스 없이 스크롤 정지
- 뒤로가기: Step 2로 이동, 이전 입력값 유지

### 5.5 Step 4 — 활동량 선택

| 레벨 | 라벨 | 활동 계수 |
|------|------|----------|
| 1 | 극소 활동 | 1.2 |
| 2 | 간헐적 활동 | 1.375 |
| 3 | 생활 밀착 활동 | 1.55 |
| 4 | 고강도 종사 | 1.725 |

- 중복 선택 불가, 재터치 시 선택 해제 → 다음 버튼 비활성화
- 뒤로가기: Step 3으로 이동, 이전 입력값 유지

### 5.6 Step 5 — 목표 선택

| 목표 | 칼로리 계수 | 단백질 상수 | 지방 비율 |
|------|-----------|-----------|----------|
| 감량 | TDEE × 0.8 | 1.6 | 0.2 |
| 유지 | TDEE × 1.0 | 1.2 | 0.25 |
| 증량 | TDEE × 1.15 | 1.8 | 0.2 |

- 중복 선택 불가, 재터치 시 선택 해제 → 다음 버튼 비활성화
- 뒤로가기: Step 4로 이동, 이전 입력값 유지

### 5.7 결과 화면 — TDEE 산출

- 로딩 상태: "입력한 정보로 목표값을 찾는 중입니다..." 표시
- 로딩 완료 후 결과 화면 페이드인
- 반도넛 차트: 권장 탄/단/지 비율을 좌→우 방향 채움 애니메이션(500ms)으로 표시
- 결과 표시 항목: 일일 권장 칼로리, 탄수화물, 단백질, 지방, 물 권장 섭취량, 당류, 나트륨, 식이섬유, 콜레스테롤
- 뒤로가기: Step 5로 이동, 이전 입력값 유지

### 5.8 시작하기 API 시퀀스 (순차 호출)

1. 시작하기 버튼 터치 → 버튼 비활성화 (중복 방지)
2. POST /api/v1/nutrition/calculate { sex, birthYear, heightCm, weightKg, activityLevel, goal }
3. POST /api/v1/member/register { memSex, memBirthDate, socialSlct, socialKey }
4. POST /api/v1/auth/login { socialSlct, socialKey } → Set-Cookie 저장
5. POST /api/v1/nutrition/register { 신체정보 + ②의 계산 결과 전체 }
6. 홈 화면 이동

### 5.9 페이지 전환 인터랙션

| 방향 | 애니메이션 |
|------|-----------|
| 전진 | 우→좌 슬라이드 (300ms) + 다음 화면 50ms 딜레이 |
| 후퇴 | 좌→우 슬라이드 (300ms) |

- 프로그레스바: N/5 진행도 게이지, 단계 이동 시 300ms ease-out 애니메이션

## 6. 에러 처리

| 에러 상황 | 복구 전략 | 사용자 피드백 | 로깅 |

| 네트워크 오류 | 버튼 재활성화 | 토스트 | warn |

## 7. 범위

**포함:**
- 온보딩 5단계 (성별/출생연도/신장·체중/활동량/목표)
- TDEE 결과 화면
- 시작하기 API 시퀀스 (영양소 계산 → 회원 등록 → 로그인 → 영양소 등록)
- 단계별 뒤로가기 및 입력값 유지

**제외:**
- 스플래시/로그인 (SPEC-001)
- 홈 화면 (SPEC-003)
- 회원 정보 수정 (온보딩 이후)

## 8. 제약

- 시작하기 버튼 API 호출 중 비활성화 (중복 가입 방지)
- 온보딩 진행 중 앱 종료 후 재진입 시 Step 1부터 재시작
- 최소 터치 영역: 48×48dp
- 픽커 스크린 리더 지원 (값 변경 시 읽어주기)
- 프로그레스바 텍스트 대안 제공 (예: "5단계 중 2단계")

## 10. 기술 설계

- 아키텍처: MVI + Clean Architecture
- OnboardingViewModel: 5단계 전체 상태를 단일 ViewModel에서 관리. 각 단계 입력값(성별/출생연도/신장/체중/활동량/목표)을 OnboardingState로 보관
- NutritionCalculateUseCase: API 시퀀스 (①→②→③→④) 처리
- 로컬 저장소: DataStore (세션 쿠키 저장)
