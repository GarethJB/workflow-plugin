# Data Model: 스플래시 / 로그인 플로우

**Feature**: 001-splash-login-flow  
**Date**: 2026-04-15

---

## 엔티티 정의

### Server (서버)

앱에서 연결할 수 있는 서비스 환경을 나타낸다.

| 필드 | 타입 | 설명 | 제약 |
|------|------|------|------|
| `name` | String | 화면에 표시되는 서버 이름 | 필수, 비어 있을 수 없음 |
| `apiUrl` | String | API 기본 주소 | 필수, 유효한 URL 형식 |
| `imgServerUrl` | String | 이미지 서버 주소 | 선택 |
| `photoUrl` | String | 프로필 사진 서버 주소 | 선택 |
| `photoUploadUrl` | String | 사진 업로드 서버 주소 | 선택 |

**상태 전이**: 없음 (읽기 전용, 서버에서 목록 제공)

**저장 위치**: 메모리 (목록) + 로컬 저장소 (선택된 서버 `apiUrl` 영속 저장)

---

### UserSession (사용자 세션)

로그인 성공 후 로컬에 저장되는 인증 상태.

| 필드 | 타입 | 설명 | 제약 |
|------|------|------|------|
| `sessionId` | String | 서버 발급 세션 ID | 필수 |
| `rememberMeToken` | String | 자동 로그인 토큰 | 선택 (서버 응답에 포함될 경우) |
| `savedAt` | DateTime | 세션 저장 시각 | 필수 (디버깅용) |

**상태 전이**:

```
[없음]
  → (로그인 성공) → [저장됨]
  → (서버 수동 변경) → [삭제됨] → [없음]
  → (세션 만료 감지) → [삭제됨] → [없음]
```

**저장 위치**: 로컬 영속 저장소 (암호화 권장)

**검증 규칙**:
- `sessionId`가 비어있는 경우 저장하지 않음
- 새 세션 수신 시 기존 값을 덮어씀 (갱신)

---

### SocialIdentity (소셜 계정)

소셜 인증 제공자와 해당 사용자 식별자.

| 필드 | 타입 | 설명 | 제약 |
|------|------|------|------|
| `provider` | Enum | 소셜 제공자 | 필수: NAVER / KAKAO / GOOGLE / APPLE |
| `token` | String | 소셜 인증 후 획득한 토큰 | 필수 |

**상태 전이**:

```
[미인증]
  → (SDK 인증 성공) → [인증됨]
  → (SDK 인증 실패) → [실패]
  → (가입 확인 완료) → [처리 완료] → (메모리에서 해제)
```

**저장 위치**: 메모리 전용 (로그인 흐름 중 일시 사용, 영속 저장 불필요)
- 단, 미등록 사용자의 경우 온보딩 화면 전환 시 전달

---

### Member (회원)

서비스에 등록된 사용자. 로그인 플로우에서 가입 여부 확인에만 사용.

| 필드 | 타입 | 설명 | 제약 |
|------|------|------|------|
| `isRegistered` | Boolean | 서비스 등록 여부 | 서버 응답으로 결정 |

**상태 전이**:

```
소셜 인증 성공 →
  isRegistered = true  → 로그인 API 호출 → 세션 저장 → 메인 화면
  isRegistered = false → 온보딩 화면 (SocialIdentity 전달)
```

## 화면 상태 모델

### SplashScreenState

| 상태 | 설명 | UI |
|------|------|-----|
| `Loading` | 서버 목록 조회 중 | 로딩 인디케이터 |
| `ServerSelection` | 저장된 서버 없음, 선택 필요 | 서버 목록 표시 |
| `AutoConnecting` | 저장된 서버로 자동 연결 중 | 로딩 인디케이터 |
| `Error` | 서버 목록 조회 실패 | 에러 메시지 |

### LoginScreenState

| 상태 | 설명 | UI |
|------|------|-----|
| `Idle` | 초기 상태 | 소셜 버튼 활성화 |
| `Loading` | 소셜 인증 처리 중 | 로딩 인디케이터, 버튼 비활성화 |
| `Error` | 인증 실패 | 에러 메시지, 버튼 재활성화 |

---

## 엔티티 관계

```
Server (목록)
  └─ 선택 → 로컬 저장 (apiUrl)
              └─ 존재 시 → UserSession 조회
                            ├─ 존재 → 메인 화면
                            └─ 없음 → LoginScreen

SocialIdentity
  └─ 인증 성공 → Member.isRegistered 확인
                  ├─ true → UserSession 생성 → 메인 화면
                  └─ false → 온보딩 화면 (SocialIdentity 전달)
```
