# UI Flow Contracts: 스플래시 / 로그인 플로우

**Feature**: 001-splash-login-flow  
**Date**: 2026-04-15

---

## 화면 전이 계약

이 문서는 스플래시/로그인 플로우의 각 화면 간 전이 조건과 전달 데이터를 정의한다.

### 진입점: 앱 실행 → 스플래시 화면

**진입 조건**: 앱이 시작되면 항상 스플래시 화면부터 시작한다.

**초기 동작**:
1. 서버 목록 조회 시작 (비동기)
2. 로컬 저장소에서 이전 서버 URL 조회

---

## 스플래시 화면 (SplashScreen)

### 책임
- 서버 목록 조회
- 저장된 서버 자동 선택 또는 서버 선택 UI 표시
- 세션 존재 여부 확인 후 다음 화면 결정

### 화면 상태

```
앱 실행
  └→ [Loading] 서버 목록 조회 중
        ├→ [Error] 서버 목록 조회 실패
        │    └→ 에러 메시지 표시 (앱 재실행으로 재시도)
        └→ 성공
              ├→ 저장된 서버 URL 존재?
              │    └→ YES: [AutoConnecting] 자동 서버 선택
              │              └→ 세션 존재? YES → 메인 화면 이동
              │                            NO  → 로그인 화면 이동
              └→ NO: [ServerSelection] 서버 목록 표시
                          └→ 사용자가 서버 선택
                                └→ 세션 삭제 + 로그인 화면 이동
```

### 화면 → 메인 화면 전이

| 조건 | 전달 데이터 | 비고 |
|------|-------------|------|
| 저장된 세션 존재 (자동 연결 시) | 없음 | 세션 유효성 검증 없이 즉시 이동 |

### 화면 → 로그인 화면 전이

| 조건 | 전달 데이터 | 비고 |
|------|-------------|------|
| 저장된 세션 없음 (자동 연결 시) | 없음 | |
| 사용자가 수동으로 서버 선택 | 없음 | 이동 전 기존 세션 삭제 수행 |

## 로그인 화면 (LoginScreen)

### 책임
- 소셜 로그인 버튼 표시 (네이버/카카오/구글/애플)
- 소셜 SDK 인증 처리
- 가입 여부 확인 후 다음 화면 결정

### 화면 상태

```
[Idle] 버튼 활성화
  └→ 소셜 로그인 버튼 탭
        └→ [Loading] 버튼 전체 비활성화
              ├→ SDK 인증 실패
              │    └→ [Error] 에러 메시지 + 버튼 재활성화 → [Idle]
              └→ SDK 인증 성공
                    └→ 가입 확인 API 호출
                          ├→ 실패: [Error] 에러 메시지 + 버튼 재활성화 → [Idle]
                          └→ 성공
                                ├→ 기존 회원:
                                │    └→ 로그인 API 호출
                                │          ├→ 실패: [Error] + 버튼 재활성화 → [Idle]
                                │          └→ 성공: 세션 저장 → 메인 화면 이동
                                └→ 미등록:
                                     └→ 온보딩 화면 이동
```

### 화면 → 메인 화면 전이

| 조건 | 전달 데이터 | 비고 |
|------|-------------|------|
| 기존 회원 로그인 성공 | 없음 | 세션은 로컬 저장 후 이동 |

### 화면 → 온보딩 화면 전이

| 조건 | 전달 데이터 | 비고 |
|------|-------------|------|
| 미등록 사용자 소셜 인증 성공 | `SocialIdentity` (provider + token) | 온보딩에서 회원가입 처리에 사용 |

---

## 메인 화면 → 로그인 화면 전이 (역방향)

| 조건 | 수행 동작 | 비고 |
|------|---------|------|
| 서버로부터 인증 실패 응답 수신 | 저장된 세션 삭제 → 로그인 화면 이동 | 스플래시 화면 경유하지 않음 |

---

## 중복 요청 방지 계약

| 화면 | 방지 대상 | 방지 방법 |
|------|-----------|-----------|
| 스플래시 | 서버 선택 중복 탭 | 3초 내 중복 요청 무시 |
| 로그인 | 소셜 버튼 중복 탭 | 처리 중 모든 버튼 비활성화 |

---

## 에러 메시지 계약

모든 에러는 화면 내 인라인 텍스트로 표시한다. 앱 재시작 없이 재시도 가능해야 한다.

| 상황 | 에러 유형 | 재시도 방법 |
|------|-----------|-------------|
| 서버 목록 조회 실패 | 서버 오류 또는 네트워크 오류 | 앱 재실행 |
| 소셜 SDK 인증 실패 | 인증 취소 또는 오류 | 버튼 재탭 |
| 가입 확인 API 실패 | 네트워크 또는 서버 오류 | 버튼 재탭 |
| 로그인 API 실패 | 네트워크 또는 서버 오류 | 버튼 재탭 |
